OpenRouter：模型聚合平台

OpenRouter 是一個統一的 API 閘道，讓你可以透過單一介面存取多家供應商與開放模型。

為什麼使用 OpenRouter？

傳統上，如果你想同時使用 GPT-5.5、Claude Sonnet 5、MiniMax M3 和 Llama 4，你需要分別向 OpenAI、Anthropic、MiniMax、Meta 或雲端供應商申請 API Key。OpenRouter 解決了這個痛點。

統一 API
使用單一 API Key 即可調用數百種模型，無需維護多個帳戶。
價格透明
OpenRouter 通常直接傳遞供應商的價格，有時甚至更便宜（透過大量採購）。
高可用性
當某個模型供應商（如 Anthropic）當機時，OpenRouter 可自動切換到其他供應商提供的相同模型。
功能 官方 API OpenRouter
Key 管理 每個廠商一個 Key 一個 Key 搞定全部
支付方式 需綁定多張信用卡 統一儲值（支援加密貨幣）
模型覆蓋 僅限該廠商 涵蓋 OpenAI, Anthropic, Google, Meta, DeepSeek, MiniMax 等
延遲 最低 (直連) 極低 (多一層轉發，體感不明顯)

我該選哪一個模型？

根據你的具體使用情境，我們給出以下建議；若經由 OpenRouter 使用，建議同時設定備援模型與成本上限。

日常全能助理
首選 GPT-5.5 或 GPT Chat Latest。適合需要穩定綜合能力、多模態與大型任務拆解的日常工作流。
程式開發 / 重度寫作
首選 Claude Sonnet 5。需要更重的長文策略、創意寫作或高品質輸出時，可改用 Claude Fable 5 或 Claude Opus 4.8。
超長文件 / 論文分析
首選 Gemini 3.5 Flash 或 Gemini 3.1 Pro Preview。適合大量文件、影片與 Google 生態整合。
開發預算有限
首選 DeepSeek V4 Flash 或 MiniMax M3。前者適合高性價比程式碼與推理，後者適合中文與長上下文代理任務。

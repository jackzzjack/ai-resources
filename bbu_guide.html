什麼是 BBU？

Battery Backup Unit — 為揮發性快取記憶體提供緊急電源的關鍵保護機制

BBU（Battery Backup Unit，電池備援單元） 是一種將可充電電池（或超級電容）與揮發性記憶體（DRAM/SRAM）組合在一起的模組。
其核心目的是：當主機電源突然中斷時，由電池持續供電，讓揮發性記憶體中的資料得以保留，待電源恢復後再安全寫入永久儲存媒體，避免資料遺失。

🎯 核心目標

保護 RAID 控制器寫入快取中尚未刷新至磁碟的資料，在非預期斷電時不遺失任何已確認的寫入。

⚙️ 組成要素

可充電鋰電池（或超級電容）+ 揮發性 DRAM 快取 + 充電電路 + 電量監控 IC，整合為單一可插拔模組。

📍 安裝位置

通常安裝於伺服器 RAID 控制卡（HBA）旁，透過排線或直接插槽連接，為控制卡上的 DRAM 快取供電。

⏱️ 保護時間

典型 BBU 可在電源中斷後持續供電 24～72 小時（依容量與快取大小而異），等待電源恢復或執行手動備份。

為何需要 BBU？

寫入快取的效能代價與風險：寫回模式的兩難困境

🐢 Write-Through（寫穿）模式

  • 每筆寫入必須等資料實際寫到磁碟才回報完成
  • 安全性最高，斷電不會遺失資料
  • 延遲高（受制於磁碟轉速或 SSD 寫入速度）
  • 無法聚合小型隨機寫入，IOPs 受限
  • 無需 BBU，適用低效能需求場景

🚀 Write-Back（寫回）模式

  • 資料寫入 DRAM 快取後即回報完成，非同步刷入磁碟
  • 延遲極低（DRAM 納秒級 vs 磁碟毫秒級）
  • 可聚合寫入、重新排序、提升吞吐量
  • 若斷電時快取尚未刷新，資料永久遺失
  • 必須搭配 BBU 才能安全啟用
關鍵矛盾：Write-Back 模式可帶來 5～10 倍的 IOPS 提升，但 DRAM 本質上是揮發性記憶體，斷電即失。 BBU 的存在正是為了彌合這道鴻溝——讓系統能安全地啟用 Write-Back，享受效能紅利，同時保有資料安全保障。

🔋 電池模組

鋰離子電池組，提供穩定的 3.7V 輸出。內建 BMS（電池管理系統）監控電壓、電流與溫度，防止過充過放。壽命約 2～3 年，需定期更換。

⚡ 充電電路

在系統正常供電時，透過 DC-DC 轉換器為電池恆流/恆壓充電。同時監控電池健康狀態（SOH），並在電池容量低於閾值時向 RAID 控制器發出警告，切換至 Write-Through 模式。

🧠 DRAM 快取

揮發性 DDR SDRAM，容量 256MB～4GB。正常工作時由系統 12V/5V 供電；偵測到斷電後，BBU 立即接管，維持 DRAM 最低自刷新電壓（通常 < 50mW）。

📊 監控 IC

即時上報電池剩餘容量（SOC）、溫度、充電週期數給 RAID 控制器。管理員可透過工具（如 MegaCLI、storcli）查詢 BBU 狀態並設定警報閾值。

與傳統記憶體的差異

揮發性 vs 非揮發性、硬體保護 vs 軟體保護的本質區別

傳統記憶體（DRAM）斷電即失；傳統非揮發性記憶體（Flash）寫入速度慢、壽命有限。 BBU 的獨特之處在於：它讓「揮發性 DRAM」在特定時間窗口內表現得像「非揮發性記憶體」，兼顧了 DRAM 的極速存取與資料持久性。

讀取延遲 ~50 ns ~100 µs ~50 ns（DRAM 速度） ~50 ns（正常運作）
寫入延遲 ~50 ns ~1 ms（含擦除） ~50 ns（DRAM 速度） ~50 ns（正常運作）
資料保護機制 無 本身即非揮發 外部電源（電池） 電容 + 自動倒入 Flash

🔴 傳統 DRAM 的弱點

  • DRAM 每個記憶格由一個電容 + 一個電晶體組成（1T1C）
  • 電容電荷會洩漏，需要每 64ms 執行一次刷新（Refresh）
  • 刷新需要供電；電源中斷，刷新停止，資料在數毫秒～數百毫秒內消失
  • 溫度越高，洩漏越快，資料存活時間更短
  • 這是 DRAM 物理特性決定的，無法純靠軟體解決

🟢 BBU 如何解決問題

  • BBU 偵測到主電源消失時，毫秒內自動切換為電池供電
  • 持續為 DRAM 提供最低自刷新所需電流（通常不超過 50mW）
  • 電池容量設計可持續供電數十小時，遠超一般停電時間
  • 電源恢復後，RAID 控制器將快取資料安全刷新至磁碟
  • 整個過程對主機完全透明，無需作業系統介入
一句話總結：DRAM 的揮發性是物理層面的天然弱點，BBU 用「持續供電」來對抗這個弱點，而不是試圖改變 DRAM 的儲存原理。這是一種以硬體備援換取效能最大化的工程權衡。

BBU 完整運作流程

正常狀態、斷電保護、電源恢復的三階段詳解

①

正常運作（系統上電）

系統電源為 RAID 控制卡供電，同時透過 BBU 充電電路維持電池在 90%+ 電量。RAID 控制器啟用 Write-Back 模式，所有寫入請求先存入 DRAM 快取，確認後立即回報主機完成，再非同步排程刷入磁碟。

②

斷電偵測（毫秒級反應）

BBU 的電源監控電路持續偵測輸入電壓。當電壓跌落超過閾值（通常低於額定值 10%），觸發中斷訊號，BBU 在 <1ms 內自動接管供電，確保 DRAM 刷新不中斷。

③

備援供電（電池維生）

電池以最小電流維持 DRAM 自刷新（DRAM Self-Refresh 模式），RAID 控制器進入低功耗狀態，停止新的 I/O 請求。快取資料靜靜等待在 DRAM 中，理論上可存活電池剩餘容量所對應的時間（24～72 小時）。

④

電源恢復（資料回寫）

主電源恢復後，RAID 控制器重新初始化，掃描 DRAM 快取中的 dirty 資料，按寫入序列依序刷新至磁碟。完成後，系統回報所有待寫入操作已安全完成，恢復 Write-Back 模式服務。

⑤

電池耗盡降級（安全回退）

若電源長時間未恢復，電池電量低於安全閾值時，RAID 控制器自動切換至 Write-Through 模式（以效能換安全），並產生 SNMP 告警通知管理員。此時快取中剩餘 dirty 資料可能遺失，但系統不會靜默損壞資料。

現代替代方案

超越傳統電池的新世代快取保護技術

⚡

FBWC / CacheVault

Flash-Backed Write Cache

使用超級電容（Supercapacitor）取代傳統鋰電池。斷電時，電容提供數秒至數分鐘電力，RAID 控制器將 DRAM 快取資料快速「傾倒」至板載 NAND Flash 中永久保存。電源恢復後再從 Flash 讀回。

超長壽命（10 年+） 免維護 依賴 Flash 耐久性
🧠

NVDIMM-N

Non-Volatile DIMM

在標準 DDR4/DDR5 DIMM 介面上整合 DRAM + NAND Flash + 超級電容。正常使用與標準 DRAM 相同速度；斷電時，超級電容驅動控制器自動將 DRAM 資料倒入 NAND，重啟後再回復。完全對 OS 透明。

標準介面相容 自動持久化 成本最高
🔥

Intel Optane / 3D XPoint

持久記憶體（PMem）

革命性的非揮發性記憶體介質，延遲接近 DRAM（~300ns），但斷電資料不遺失。可作為持久記憶體（PMem）直接掛載為記憶體層，無需電池或電容即可同時達成高速與持久。（已於 2022 年停產）

真正非揮發 無需電池 已停產
💾

NVMe SSD 寫入快取

基於 NVMe SSD 的分層快取

以 NVMe SSD 作為 HDD 陣列的快取層（如 ZFS L2ARC、bcache、dm-cache）。NVMe 本身非揮發，無需電池保護；但延遲仍高於 DRAM。適合讀取快取場景，寫入快取需搭配掉電保護（Power Loss Protection）電容。

軟體定義 容量可擴充 延遲高於 DRAM
方案 保護機制 備援時間 維護成本 效能影響
傳統 BBU（鋰電池） 持續供電 DRAM 24～72 小時 每 2～3 年換電池 無
FBWC（超級電容） 電容 + 傾倒至 Flash 數秒（足夠倒資料） 幾乎免維護 斷電後重啟稍慢
NVDIMM-N 超電容 + 自動倒 Flash 數秒（自動完成） 低 與 DRAM 相同

應用場景

BBU 在各產業中的實際部署

🖥️

企業伺服器 / 資料中心

最主要應用場景

企業級 RAID 控制卡（如 LSI MegaRAID、HPE Smart Array、Dell PERC）標配 BBU 或 FBWC，保護資料庫（Oracle、MySQL）的 Write-Back 快取，提升交易 IOPS 同時確保 ACID 持久性。

RAID 5/6/10 資料庫保護
🏭

工業控制系統（PLC）

BBSRAM 應用

PLC（可程式邏輯控制器）使用電池備援 SRAM（BBSRAM）保存程式碼與即時運行資料，即使斷電也不中斷控制邏輯記憶。工廠自動化、機器人控制、電力調度等關鍵場景廣泛使用。

工業可靠性 即時控制
🌐

網路設備

路由器 / 交換器

高階路由器與核心交換器使用 BBU 保護轉發表（FIB）、BGP routing table 與配置資料。斷電重啟後無需重新收斂路由，大幅縮短網路中斷時間（RTO）。

快速恢復 路由保護
🚗

嵌入式 / 車用系統

BBSRAM / RTC 應用

汽車 ECU（引擎控制單元）使用電池備援 SRAM 保存學習值（如怠速學習、爆震修正），點火熄滅後記憶不消失。RTC（即時時鐘）IC 內建微型電池，維持時間計時。

車用可靠性 ECU 學習值
BBU vs UPS 的區別：UPS（不斷電系統）保護的是整台伺服器的電源，範圍廣但成本高；BBU 只保護 RAID 控制器上的快取記憶體，成本極低（數十美元）卻能精準解決寫入快取的資料安全問題。兩者並非替代關係，高可用環境通常同時部署。

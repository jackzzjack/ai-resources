GPT Partition Attribute：
64-bit flags 不是魔法開關

它是寫在每個 GPT partition entry 裡的一串旗標；真正決定它有沒有作用的，是 firmware、bootloader、作業系統或 installer 之間的共同約定。

一個 entry誰會讀它從 bit 到行為位元地圖契約控制台安全清單

一眼看懂：Attribute 只是五欄資料的其中一欄

把 GPT partition entry 想成一張設備身分證。名稱給人看，範圍描述它在哪裡，GUID 描述它是誰；attributes 則是留給能讀懂它的元件使用的旗標欄位。

GPT PARTITION ENTRY128 bytes（常見配置）
🏷️Type GUID

這一類分割區的通用角色，例如 EFI System Partition。

🪪Unique GUID

這一個 entry 的身分；在 Linux 常可對應 PARTUUID。

📍Start / End LBA

實體位置與大小，而不是檔案系統內容。

⚑Attributes

64 個 bit；只有讀取端定義其意義時才有作用。

✎Name

人類可讀的 label；不等於 Type GUID 或 flag。

先懂這個：attribute 不是檔案系統 mount option，也不是 partition name。把某個 bit 設成 1，並不會「自動」讓系統採取動作。

UEFI / firmware

可能依 Type GUID 與平台規則辨識可開機項目；不是每個 attribute 都由它解讀。

Bootloader

A/B slot 設計常在這層定義 active、successful、tries-left 等自己的狀態契約。

Installer / OS

負責建立與更新 layout，或讓管理工具能觀察、驗證與修復狀態。

常見誤解：「這個 flag 看起來沒被 Linux 用到」不等於它安全可清除；它可能是下一階段 bootloader 或復原流程的輸入。

從一個 bit 到實際行為：需要一條完整的契約鏈

01

定義

團隊文件說明哪一個 bit 代表什麼、誰能改、何時清除。

02

寫入

installer 或工具修改 GPT metadata，並保留備份與版號資訊。

03

讀取

bootloader / OS 在正確時機讀取指定 bit。

04

決策

讀取端依契約選擇 slot、限制嘗試次數或決定後續流程。

05

回寫

成功或失敗時更新狀態，並由測試矩陣驗證升級與 rollback。

危險點：若只有「寫入」沒有「讀取」，flag 只是資料；若讀取端與寫入端對 bit 的意義不同，才會變成難以重現的開機問題。

64-bit 位元地圖：先分清共通區、保留區與類型專屬區

UEFI GPT 規範為 attributes 留出 64 個位元。部分低位元具有通用定義；中間區段保留；高位元可由分割區 Type GUID 的規範定義。實際使用前應查目標 Type GUID 與平台文件。

區段怎麼理解實務提醒
bit 0–2規範定義的通用 attributes，例如 required、no block I/O protocol、legacy BIOS bootable。仍須確認你的平台是否實作、何時讀取。
bit 3–47保留區。不要把「尚未用到」當作可任意挪用的空間。
bit 48–63由 partition Type GUID 的相關規範定義的 type-specific attributes。沒有查 Type GUID 文件，就不應猜測高位元的語意。

契約控制台：換一個讀取者，結果就不同

選擇你以為正在設定的情境。這個小模擬器刻意不直接給「一行指令」，而是先要求確認誰寫、誰讀與怎麼驗證。

你想用 attribute 解決什麼？

先檢查，再改：把 GPT 當成可回復的部署資料

1. 盤點 layout
sudo sgdisk -p /dev/sdX
確認磁碟、entry 編號、Type GUID 與整體配置。
2. 檢視單一 entry
sudo sgdisk -i 3 /dev/sdX
先讀出現況，別從記憶猜 bit 是否已被使用。
3. 備份與記錄
先備份 GPT metadata；記錄 layout version、bit 擁有者、語意與修改時間。
4. 驗證完整流程
在測試環境覆蓋首次安裝、升級、rollback、掉電與多次開機，而不是只驗證單次成功。
# 修改前：先做只讀檢查
sudo sgdisk -p /dev/sdX
sudo sgdisk -i 3 /dev/sdX

# 使用 sgdisk -A 前：確認本機 gdisk 版本的語法，
# 以及目標平台 / Type GUID 對 attribute 的明確規範。
你可以這樣想：Type GUID 告訴你「這一格大致是什麼」，attribute 是一張附註便條；只有拿著共同說明書的人，才知道便條上的某個勾選代表什麼。真正可靠的不是 flag 本身，而是可測試、可版本化的 contract。

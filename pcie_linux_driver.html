什麼是 PCIe（PCI Express）
PCIe（Peripheral Component Interconnect Express） 是現今最主流的高速串列匯流排介面標準，由 PCI-SIG 制定並持續演進。它取代了早期的平行式 PCI / AGP 介面，以點對點串列連線、差分訊號、全雙工傳輸為核心，提供極高頻寬與低延遲，廣泛用於 GPU、NVMe SSD、網路卡（NIC）、AI 加速器等高效能元件。
串列 + 點對點
每條 PCIe 連線（Link）由一或多條 Lane 組成，每條 Lane 含 4 條差分線（TX+/TX−、RX+/RX−），實現全雙工通訊，不再有舊式 PCI 的匯流排競爭問題。
Lane 可擴展
支援 x1、x2、x4、x8、x16、x32 等 Lane 組態；Lane 數愈多頻寬愈大。GPU 常用 x16，NVMe SSD 常用 x4，M.2 網卡用 x1。
分層架構
PCIe 協定分為三層：
① 交易層（Transaction Layer）
② 資料鏈路層（Data Link Layer）
③ 實體層（Physical Layer）
類似網路 OSI 模型的概念設計。
向下相容
PCIe 插槽與裝置向下相容；PCIe 5.0 裝置可插入 4.0 插槽，以較低速協商運作。物理插槽也可支援較小 x 數（x16 槽接 x4 卡）。
記憶體映射 I/O
PCIe 裝置的暫存器與記憶體透過 BAR（Base Address Register）映射至 CPU 的實體位址空間，驅動程式可直接讀寫，無需特殊 I/O 指令。
MSI / MSI-X 中斷
取代舊式 INTx 共享中斷，透過記憶體寫入觸發中斷，支援多向量（MSI-X 最多 2048 個），大幅提升多核心環境的中斷分散效率。

拓撲架構（Topology）
Root Complex（RC）
PCIe 拓撲的根節點，由 CPU 或 PCH 提供。負責管理位址空間分配（BARs）、產生/接收 TLP、連接 DRAM 與 PCIe Fabric。Linux 中對應 /sys/bus/pci/devices/ 的 domain:bus 根。
PCIe Switch
提供多個下游埠（Downstream Port），讓一個上游連線可扇出至多個 Endpoint。Switch 本質是一個中繼路由器，依 Bus Number 轉發 TLP，在 AI 伺服器與 NVMe JBOF 中大量使用。
Endpoint（EP）
實際的功能裝置（GPU、NIC、NVMe SSD 等），是 TLP 的最終來源或目的地。EP 可以是 Type 0 Header（不轉發）。SR-IOV 技術可讓一個 Physical Function 虛擬出多個 Virtual Function（VF）。
📦
TLP 封包協定（Transaction Layer Packet）
PCIe 三層協定簡介 交易層（TL） → 產生/消費 TLP，處理流量控制 ｜ 資料鏈路層（DLL） → DLLP、ACK/NAK、重傳 ｜ 實體層（PL） → 8b/10b 或 128b/130b 編碼、時脈、差分訊號
TLP 主要類型
MRd / MWr — 記憶體讀寫（最常用，Driver BAR 存取）
IORd / IOWr — Legacy I/O 讀寫（舊式相容）
CfgRd / CfgWr — 組態空間讀寫（RC 枚舉用）
Msg — 訊息（INTx 模擬、錯誤報告、電源管理）
Cpl / CplD — 完成封包（帶或不帶資料的回應）
TLP Header 結構（MWr 為例）
Fmt[2:0]封包格式（有無 Data、32/64b 位址） Type[4:0]TLP 類型碼 TC[2:0]流量等級 Traffic Class Length[9:0]Payload 長度（DW 數） Requester IDBus:Device:Function Tag[9:0]交易標籤（用於配對 Cpl） Address[63:0]目標位址（32 或 64 位元）
DMA（Direct Memory Access）與 IOMMU PCIe 裝置可透過 DMA 直接讀寫 Host 記憶體（發出 MRd/MWr TLP 至 Host 位址）。Linux 的 IOMMU（Intel VT-d / AMD-Vi）會建立 I/O 頁表，限制裝置只能存取授權記憶體範圍，防止惡意裝置攻擊（DMA Attack）。驅動程式透過 dma_alloc_coherent() 分配可被裝置存取的 DMA 緩衝區。

Linux PCIe Driver 驅動流程
枚舉流程概述 系統啟動時，BIOS/UEFI 或 Linux 的 PCI 核心透過讀寫組態空間（Configuration Space）掃描所有匯流排，發現每個裝置的 Vendor ID / Device ID，分配 BAR 位址與 Bus Number，再由 Driver 核心比對 pci_device_id 表進行 probe。
1

系統啟動：BIOS / ACPI 枚舉

BIOS/UEFI 掃描 PCIe 拓撲，對每個 Endpoint 分配 Bus:Device:Function（BDF）、BAR 位址範圍（MMIO 與 I/O）、中斷路由。相關資訊透過 ACPI MCFG 表傳遞給 Linux，讓核心取得 ECAM（Enhanced Config Access Mechanism）基底位址。

2

PCI Core 掃描：pci_scan_bus()

Linux PCI 核心呼叫 pci_scan_bus() → pci_scan_slot() → pci_scan_device()，讀取每個裝置的 Configuration Space Header（256B ~ 4KB ECAM），建立 struct pci_dev，解析 Capability List（MSI、PCIe Capability、AER 等）。

3

Driver 比對：pci_device_id 表

每個 PCIe 驅動程式透過 MODULE_DEVICE_TABLE(pci, ...) 宣告支援的 {vendor, device, subvendor, subdevice, class, class_mask} 組合。核心的 pci_bus_match() 函式將掃描到的裝置與已載入驅動的 ID 表逐一比對。

4

Driver Probe：driver→probe(pdev, id)

比對成功後核心呼叫驅動的 .probe() 回呼。驅動程式依序：pci_enable_device() 啟用裝置 → pci_request_regions() 聲明 BAR → pci_iomap() / ioremap() 映射暫存器 → 設定 DMA mask → 配置 MSI/MSI-X 中斷 → 初始化硬體。

5

中斷配置：MSI / MSI-X

呼叫 pci_alloc_irq_vectors(pdev, min, max, PCI_IRQ_MSI | PCI_IRQ_MSIX) 配置向量，再用 request_irq() 或 request_threaded_irq() 註冊 ISR。NVMe 與高速 NIC 通常為每個 CPU core 配置一個 MSI-X 向量實現中斷親和性。

6

DMA 設定：Coherent / Streaming

dma_set_mask_and_coherent(dev, DMA_BIT_MASK(64)) 設定裝置可定址的記憶體範圍。dma_alloc_coherent() 分配 CPU 與裝置均可見的一致性緩衝區（適合 descriptor ring）；dma_map_single() / dma_map_sg() 用於 streaming 模式（資料路徑）。

7

裝置移除：driver→remove(pdev)

驅動程式必須按照 probe 的反序釋放資源：停止硬體 → free_irq() → pci_free_irq_vectors() → dma_free_coherent() → pci_iounmap() → pci_release_regions() → pci_disable_device()。任何資源洩漏均會導致 kernel oops 或記憶體洩漏。

8

電源管理：PCIe ASPM / D-states

驅動可實作 struct dev_pm_ops（.suspend/.resume）支援 S3/S4 睡眠。PCIe ASPM（L0s/L1/L1ss）由核心 drivers/pci/pcie/aspm.c 管理；pci_save_state() / pci_restore_state() 保存/恢復組態空間。

PCIe Driver 核心 API 與程式碼範本
#include <linux/pci.h> #include <linux/module.h> /* 宣告支援的裝置列表 */ static const struct pci_device_id my_pci_ids[] = { { PCI_DEVICE(0x10DE, 0x2684) }, /* NVIDIA RTX 4090 */ { PCI_DEVICE(0x1022, 0x149C) }, /* AMD Instinct MI300X */ { 0, } }; MODULE_DEVICE_TABLE(pci, my_pci_ids); static int my_probe(struct pci_dev *pdev, const struct pci_device_id *id) { int ret; ret = pci_enable_device(pdev); /* 1. 啟用裝置電源 */ if (ret) return ret; ret = pci_request_regions(pdev, "my_driver"); /* 2. 聲明 BAR */ if (ret) goto err_disable; pci_set_master(pdev); /* 3. 允許 Bus Master（DMA） */ ret = dma_set_mask_and_coherent(&pdev->dev, DMA_BIT_MASK(64)); if (ret) goto err_release; /* 4. 設定 64-bit DMA mask */ /* 5. 映射 BAR、配置 MSI-X、初始化硬體 … */ return 0; err_release: pci_release_regions(pdev); err_disable: pci_disable_device(pdev); return ret; } static void my_remove(struct pci_dev *pdev) { /* 按 probe 反序釋放資源 */ pci_release_regions(pdev); pci_disable_device(pdev); } static struct pci_driver my_driver = { .name = "my_pcie_driver", .id_table = my_pci_ids, .probe = my_probe, .remove = my_remove, }; module_pci_driver(my_driver);
/* ── BAR（Base Address Register）映射 ── */ /* 取得 BAR 0 的實體位址與大小 */ resource_size_t bar_start = pci_resource_start(pdev, 0); resource_size_t bar_len = pci_resource_len(pdev, 0); /* 映射至虛擬位址（ioremap 內部也做 barrier） */ void __iomem *bar = pci_iomap(pdev, 0, bar_len); if (!bar) return -ENOMEM; /* 讀寫裝置暫存器（使用 accessor，保證記憶體順序） */ u32 val = ioread32(bar + 0x100); /* 讀 32-bit 暫存器 */ iowrite32(0xDEADBEEF, bar + 0x104); /* 寫 32-bit 暫存器 */ /* 完成後解除映射 */ pci_iounmap(pdev, bar); /* ── MMIO vs I/O Port ── */ /* 判斷 BAR 類型 */ if (pci_resource_flags(pdev, 0) & IORESOURCE_MEM) { /* MMIO BAR → 使用 ioremap / ioread/iowrite */ } else if (pci_resource_flags(pdev, 0) & IORESOURCE_IO) { /* I/O Port BAR → 使用 inb/outb (已罕見) */ }
/* ── Coherent DMA（descriptor ring 常用） ── */ dma_addr_t dma_handle; void *cpu_addr = dma_alloc_coherent(&pdev->dev, PAGE_SIZE, /* 分配大小 */ &dma_handle, /* 輸出：裝置可見的物理位址 */ GFP_KERNEL); /* cpu_addr — CPU 端虛擬位址 */ /* dma_handle — 寫入裝置 descriptor 的 DMA 位址 */ /* 釋放 */ dma_free_coherent(&pdev->dev, PAGE_SIZE, cpu_addr, dma_handle); /* ── Streaming DMA（資料路徑，效能較高） ── */ dma_addr_t dma_addr = dma_map_single(&pdev->dev, buf, /* CPU 虛擬位址 */ buf_len, DMA_TO_DEVICE); /* 或 DMA_FROM_DEVICE */ if (dma_mapping_error(&pdev->dev, dma_addr)) { /* 錯誤處理 */ } /* 裝置 DMA 完成後 unmap（同步 cache） */ dma_unmap_single(&pdev->dev, dma_addr, buf_len, DMA_TO_DEVICE); /* ── Scatter-Gather DMA ── */ int nents = dma_map_sg(&pdev->dev, sgl, sg_count, DMA_FROM_DEVICE); for_each_sg(sgl, sg, nents, i) { desc->addr = sg_dma_address(sg); desc->len = sg_dma_len(sg); } dma_unmap_sg(&pdev->dev, sgl, sg_count, DMA_FROM_DEVICE);
/* ── MSI-X 中斷配置 ── */ int nvec = pci_alloc_irq_vectors(pdev, 1, /* min vectors */ 32, /* max vectors */ PCI_IRQ_MSIX | PCI_IRQ_MSI); if (nvec < 0) return nvec; /* 為每個向量註冊 ISR */ for (int i = 0; i < nvec; i++) { int irq = pci_irq_vector(pdev, i); ret = request_irq(irq, my_irq_handler, 0, "my_driver", priv); if (ret) goto err_free_vectors; } /* ISR 範例：取得 queue index 並 schedule NAPI / tasklet */ static irqreturn_t my_irq_handler(int irq, void *data) { struct my_queue *q = data; napi_schedule(&q->napi); /* 或 tasklet_schedule() */ return IRQ_HANDLED; } /* 釋放 */ for (int i = 0; i < nvec; i++) free_irq(pci_irq_vector(pdev, i), priv); pci_free_irq_vectors(pdev); /* ── 設定 IRQ affinity（綁定 CPU core）── */ irq_set_affinity_hint(pci_irq_vector(pdev, i), cpumask_of(i));

SR-IOV（Single Root I/O Virtualization）▼
SR-IOV 允許一張 PCIe 網卡或 GPU 向 OS 呈現多個 Virtual Function（VF），每個 VF 可直通給虛擬機器（VM），享有接近裸機的 I/O 效能。

啟用 SR-IOV：echo 4 > /sys/bus/pci/devices/0000:01:00.0/sriov_numvfs

驅動端需實作 pci_driver.sriov_configure()，核心呼叫時驅動程式創建 VF 的 pci_dev 並初始化其資源。VF 由獨立的輕量驅動（如 iavf、mlx5_core）管理。
AER（Advanced Error Reporting）▼
PCIe AER 是硬體層錯誤偵測與報告機制，分為 Correctable（可糾正）、Uncorrectable Non-Fatal、Uncorrectable Fatal 三類。

Linux 的 drivers/pci/pcie/aer.c 攔截 AER 中斷，解析錯誤狀態暫存器，並呼叫驅動程式的錯誤回呼（pci_error_handlers.error_detected()）。驅動可選擇重置裝置、標記錯誤或發出 kernel warning。

查看 AER 事件：dmesg | grep -i "aer\|pcie error"
手動觸發 AER 測試：setpci -s BDF ECAP_AER+0x08.l=0x4
VFIO — 用戶空間裝置直通▼
VFIO（Virtual Function I/O）框架讓用戶空間程式（如 QEMU）可安全地直接控制 PCIe 裝置，不需要 kernel driver。核心流程：

1. 解除 kernel driver：echo "0000:01:00.0" > /sys/bus/pci/drivers/nvidia/unbind
2. 綁定 vfio-pci：echo "10de 2684" > /sys/bus/pci/drivers/vfio-pci/new_id
3. QEMU 透過 /dev/vfio/<group> 取得裝置控制權，搭配 IOMMU 隔離確保安全性。

DPDK 也使用 VFIO 或 UIO 旁繞 kernel 網路 stack 達到線速封包處理。

PCIe 偵錯與診斷工具
lspci — 列出 PCIe 裝置
# 列出所有裝置（含廠商名稱）
lspci -v
# 顯示特定裝置詳細資訊與 capabilities
lspci -vvv -s 01:00.0
# 查看 PCIe Link Speed / Width
lspci -vvv | grep -A5 "LnkCap\|LnkSta"
# 以樹狀顯示拓撲
lspci -t -v
# 顯示 Kernel driver in use
lspci -k
sysfs — 運行時資訊
# 列出所有 PCIe 裝置
ls /sys/bus/pci/devices/
# 查看裝置的 Vendor / Device ID
cat /sys/bus/pci/devices/0000:01:00.0/vendor
# 查看目前使用的 driver
readlink /sys/bus/pci/devices/0000:01:00.0/driver
# 動態解除/重新綁定 driver
echo "0000:01:00.0" > /sys/bus/pci/drivers/nvme/unbind
echo "0000:01:00.0" > /sys/bus/pci/drivers/nvme/bind
setpci — 讀寫組態空間
# 讀取 Command Register（0x04）
setpci -s 01:00.0 COMMAND
# 啟用 Bus Master bit
setpci -s 01:00.0 COMMAND=0007
# 讀取 PCIe Link Status（ECAP）
setpci -s 01:00.0 ECAP_EXP+0x12.w
# 讀取 BAR0 位址
setpci -s 01:00.0 BASE_ADDRESS_0
dmesg / perf — 核心日誌與效能
# 查看 PCIe 初始化與 AER 錯誤
dmesg | grep -i "pci\|pcie\|aer"
# 動態追蹤 PCI probe
echo 'p:probe pci_device_probe' > /sys/kernel/debug/tracing/kprobe_events
# PCIe 效能計數器（pmem / GPU）
perf stat -e uncore_imc/cas_count_read/ -a sleep 1
# 查看 IRQ 分佈
cat /proc/interrupts | grep nvme
常見問題排查 Link 降速（x16 跑 x8）：確認插槽 PCIe Lane 供給、CPU PCIe Lane 數限制（多顯卡分配）。
Gen 5 → Gen 3 降版：檢查 lspci -vvv 的 LnkSta vs LnkCap，可能是 BIOS 設定或訊號品質問題。
裝置未被偵測：確認 BIOS 中 PCIe 插槽已啟用；用 lspci -nn 確認 Vendor/Device ID；排查電源是否到位（6-pin / 8-pin）。
AER Correctable Error：少量可忽略，大量需更換線材或降低 PCIe 速度（ASPM 問題）。

什麼是 SONiC？

SONiC 是一款基於 Linux 的開源網路作業系統（NOS），最初由 Microsoft 為 Azure 資料中心開發，現已貢獻至 OCP（Open Compute Project）並由 Linux Foundation 管理。

核心價值：軟硬體解耦（Disaggregation）

傳統網路設備的軟硬體是綑綁的（例如 Cisco 設備配 IOS）。SONiC 的出現打破了這種限制，讓同一套作業系統可以運行在不同廠商（Broadcom, NVIDIA, Barefoot）的交換晶片上，大幅降低了供應商鎖定（Vendor Lock-in）的風險。

🐳
容器化設計
所有網路模組均在獨立的 Docker 容器中運行，可單獨升級或重啟而不影響整體系統穩定性。
🔌
多晶片支援
透過 SAI（Switch Abstraction Interface）技術，支援市場上幾乎所有主流的 ASIC 晶片。
📈
超大規模驗證
經過 Microsoft Azure、Alibaba、Tencent 等頂尖雲端供應商的大規模生產環境驗證。
特性 傳統 NOS SONiC
軟硬體關係 緊密綑綁（Proprietary） 完全解耦（Disaggregated）
開發速度 取決於硬體廠商釋放更新 開源社群快速疊代
可客製化程度 低（僅限 CLI/API） 極高（可修改源碼、自定義容器）
維運模型 各家獨有 CLI 現代化 DevOps / Infrastructure as Code

模組化架構詳解

SONiC 的架構以「狀態分離」為核心，利用中心化的 Redis 資料庫來管理系統狀態，達成極高的靈活性與故障恢復能力。

Redis 資料庫
SONiC 的「心臟」。所有應用程式不直接通信，而是透過 Redis 進行狀態交換（Publish/Subscribe）。這使得個別進程重啟後能迅速從資料庫恢復狀態。
AppDB ConfigDB ASIC_DB
SAI 介面
Switch Abstraction Interface。它定義了一套標準的 API，讓上層軟體無需關心底層 ASIC 的具體指令集，達成「一次開發，處處運行」。
Silicon Independence C API
SWSS 與 Syncd
SWSS 負責處理網路邏輯與資料庫狀態；Syncd 則負責將資料庫中的網路配置，透過 SAI 轉化為硬體 ASIC 的具體寄存器設定。

為 AI 算力集群而生

現代 AI 訓練（LLM）需要極高的東西向（East-West）頻寬與極低的延遲，SONiC 提供了一系列關鍵特性來支撐高效能計算環境。

🚀
RoCEv2 支援
RDMA over Converged Ethernet。SONiC 完整支援 RoCEv2，讓 GPU 之間能以零拷貝（Zero-copy）方式直接交換資料，繞過 CPU 損耗。
RDMA PFC ECN
⚖️
精細流量控制
支援 PFC（優先級流量控制）與 ECN（顯式擁塞通知），防止在 400G/800G 高速環境下發生丟包，這對 AI 訓練的穩定性至關重要。
🧭
自適應路由 (AR)
在大型 AI 拓撲中，SONiC 可配合部分硬體支援自適應路由，根據網路路徑的即時負載動態調整封包路徑，避免單一鏈路擁塞。

SONiC 在 Blackwell 世代的角色

隨著 NVIDIA GB200 等新世代 AI 伺服器的普及，數據中心開始部署 800G 甚至 1.6T 網路。SONiC 的開源特性讓大型雲端廠商能針對 AI 流量模式（如 All-Reduce）開發自定義的監控與優化邏輯。

記憶體是什麼？
Memory — 電腦儲存資訊的核心元件

基本定義

記憶體（Memory）是能夠儲存二進位資訊（0 與 1）的電子元件。每個儲存單元保存一個位元（bit）， 藉由物理狀態（電荷、磁場、電阻等）的差異來代表不同的值。 記憶體依據揮發性分為兩大類： 斷電後資料消失（揮發性，如 SRAM、DRAM）， 或斷電後資料保留（非揮發性，如 NOR Flash、NAND Flash）。

6 電晶體（6T）單元結構

SRAM 最常見的是由 6 個 MOSFET 組成的儲存單元，形成一個「雙穩態」（bistable）閂鎖電路。

Q / Q̄：互補的雙穩態節點（存「1」時 Q=high, Q̄=low）。
WL（Word Line 字元線）：高電位時打開存取電晶體 AL / AR。
BL / BL̄（Bit Line 位元線）：用於讀取差分電壓或寫入數值。
PL / PR（PMOS 負載）：維持 VDD 供電，強化保持能力。
NL / NR（NMOS 驅動）：拉低對應節點至 GND。

為什麼 SRAM 不需要刷新？
6T 雙穩態結構依靠兩個互鎖的反向器持續互相驅動，只要有電源就能永遠保持狀態，不像 DRAM 的電容會慢慢漏電。

1T1C 儲存原理

DRAM 每個位元只需 1 個電晶體 + 1 個電容，面積是 SRAM（6T）的 1/6，因此能做到更大容量、更低成本。 電容充電 → 代表「1」；放電 → 代表「0」。

缺點：電容會因漏電緩慢放電，必須在資料消失前定期「刷新（Refresh）」，典型週期為每 64ms 刷新全部列。

NOR vs NAND 架構差異

NOR Flash 的每個儲存單元（浮閘電晶體）並聯連接到位元線，如同 CMOS 中的「NOR 閘」。 這使得任意位址可獨立隨機讀取，非常適合直接執行程式碼（XIP, eXecute In Place）。


寫入前必須先以扇區（Sector）為單位進行擦除（通常 64KB～128KB），無法以小單位寫入。 擦寫次數有限（約 10 萬次），且容量比 NAND 小、成本較高。

為什麼 BIOS 用 NOR 而不用 NAND？
CPU 上電後必須立刻從固定位址執行指令，NOR Flash 的隨機存取特性讓 CPU 可直接定址讀取，NAND 則需先複製到 SRAM 才能執行。

NAND 的「先擦再寫」限制：
NAND 只能將位元從 1 → 0（程式化），不能直接覆寫。要重新寫入必須先以整個 Block 為單位擦除（將所有位元設回 1），這就是為什麼 SSD 需要垃圾回收（Garbage Collection）與 TRIM 機制。

蛇模式（Snake Bit-Line）
NAND Flash 位元線折返架構 · 節省面積的關鍵設計

蛇模式的由來

在 NAND Flash 的記憶體陣列中，每條位元線（Bit Line）底部都有一個感測放大器（Sense Amplifier）。 如果每條位元線都獨立配置一個感測放大器，晶片面積會非常大。


蛇模式（Snake / Folded Bit-Line 架構）的解法是： 讓奇數位元線往下走到底部後，「折回」往上延伸，再與偶數位元線共享一個感測放大器。 兩條位元線形成一對「差動對（Differential Pair）」，狀如蛇形 S 路徑，因此得名。

蛇模式在 3D NAND 中的演化：
現代 3D NAND（如 BiCS FLASH、V-NAND）將記憶體層垂直堆疊，蛇形位元線仍然被保留作為感測放大器共用策略，但路徑從平面延伸成垂直柱狀結構。

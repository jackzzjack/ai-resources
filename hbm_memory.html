HBM 高頻寬記憶體

從 3D 堆疊原理到世代演進，深解 HBM 如何突破傳統記憶體的頻寬瓶頸，成為 AI 時代 GPU 的核心

什麼是 HBM？

High Bandwidth Memory（高頻寬記憶體）是 AMD 與 SK Hynix 於 2013 年聯合定義的新一代記憶體標準，2015 年首款採用 HBM 的 GPU — Fiji — 正式量產上市。

HBM 的核心理念：把 DRAM 晶粒垂直堆疊（3D Stacking），再透過超寬匯流排（1024-bit 起跳）直接連到 GPU / CPU 的矽中介層（Silicon Interposer）旁，大幅縮短訊號路徑、降低功耗，同時將頻寬推升到傳統 GDDR 的數倍乃至十數倍。

為什麼 AI 需要 HBM？

大型語言模型（LLM）在推理時，模型權重需從記憶體不斷搬入 GPU 計算核心，記憶體頻寬往往才是效能瓶頸，而非算力。

記憶體牆（Memory Wall）
GPU 算力每兩年翻 2–4 倍，但傳統 DRAM 頻寬只成長 1–1.5 倍，造成 GPU 「吃不飽」的局面。HBM 透過 3D 堆疊與超寬匯流排，在合理面積與功耗下，把頻寬提升到與算力相稱的水準。
實際案例：NVIDIA H100 vs. A100
H100 SXM（HBM3）頻寬 3.35 TB/s，A100（HBM2e）頻寬 2.0 TB/s，Transformer 類工作負載的 token 生成速度約提升 2–3 倍。而 H200（HBM3E）頻寬再升至 4.8 TB/s，更進一步解決推理瓶頸。

矽中介層同時承載 GPU 與多顆 HBM 堆疊，透過極短的高密度金屬線互連

① Through-Silicon Via（TSV）垂直穿孔

TSV 是一種在矽晶圓上垂直鑽孔並填充金屬（通常為銅）的技術。HBM 的每層 DRAM 晶粒都有數千個 TSV，讓訊號直接從頂層穿透到底部的 Logic Die，無需繞道傳統封裝的外部引線。
TSV 直徑~5–10 µm（遠小於傳統封裝引線）
每顆 HBM 的 TSV 數量數千至上萬個，構成超寬並行匯流排
信號延遲垂直距離極短（< 0.1 mm），延遲遠低於 PCB 走線
功耗優化短距訊號不需要強驅動電路，I/O 電壓可降低至 1.2V 以下

② 超寬介面匯流排

每顆 HBM 有 1024 條並行資料線（HBM2/3 為 1024-bit per stack），分成 8 個獨立通道（channel），每個通道 128-bit。相比之下，一顆 GDDR6 顆粒只有 32-bit 介面寬度，即使 GPU 接 12 顆 GDDR6 也只達 384-bit。

③ CoWoS 矽中介層封裝

Chip on Wafer on Substrate（CoWoS）是台積電主要的 2.5D 封裝技術：GPU 晶片與多顆 HBM 堆疊一同放置在同一片矽中介層上，透過細密金屬內連線（pitch 約 0.4 µm）傳遞超寬頻寬資料，同時大幅縮短訊號路徑至毫米等級。

HBM 之所以不像 GDDR 那樣焊在 PCB 上，是因為矽中介層的連線密度遠高於 PCB（相差 100 倍以上），這是實現 1024-bit 介面的必要條件。

HBM 的限制與挑戰

成本高昂：CoWoS 封裝製程複雜，良率低，一顆 H100 的 8 個 HBM3 堆疊成本估計超過整體 BOM 的 30–40%。

容量受限：HBM 因面積限制，目前單顆最大 36 GB（HBM3E 12-Hi），系統容量遠不如 DDR5 DIMM。

不可更換：HBM 與 GPU 封裝整合，無法像 DIMM 般升級或維修，增加維護複雜度。

供應鏈集中：目前 HBM 生產主要由 SK Hynix、Samsung、Micron 三家壟斷，供貨緊缺時直接限制 AI 加速器出貨量。

HBM 世代演進

從 2015 年 HBM1 問世至今，每一世代都大幅提升頻寬、容量與能源效率。

HBM1
2015

第一代 HBM — 概念驗證

由 SK Hynix 與 AMD 合作，首次於 AMD Fiji GPU（Radeon R9 Fury X）商用化。採用 4-Hi 堆疊（4 層），確立了 1024-bit / stack 的寬匯流排架構。

頻寬 128 GB/s / stack 堆疊 4-Hi 容量 1 GB / stack 介面 1024-bit 代表 GPU AMD Fiji
HBM2
2016

第二代 HBM — 進入 HPC 主流

JEDEC 標準化，容量翻倍（8-Hi 堆疊），頻寬大幅提升，進入 NVIDIA P100（Pascal）、AMD Vega 等高效能計算產品。成為深度學習加速器的標配。

頻寬 256 GB/s / stack 堆疊 8-Hi 容量 8 GB / stack 代表 GPU NVIDIA P100 · AMD Vega
HBM2E
2020

HBM2E — 頻寬與容量雙提升

強化版 HBM2，頻率提高至 3.6 Gbps，容量最大 16 GB / stack（16-Hi）。搭載於 NVIDIA A100、AMD MI250 等，成為主流 AI 訓練加速器。

頻寬 460 GB/s / stack 堆疊 8-Hi / 16-Hi 容量 8–16 GB / stack 代表 GPU NVIDIA A100 · AMD MI250
HBM3
2022

HBM3 — AI 訓練新標準

JEDEC 定義 HBM3，每 stack 頻寬突破 800 GB/s，容量支援至 24 GB，通道數倍增（16 channels），引入偽通道（pseudo-channel）提升效率。NVIDIA H100 SXM 為代表產品。

頻寬 >800 GB/s / stack 堆疊 12-Hi 容量 16–24 GB / stack 通道 16 channels 代表 GPU NVIDIA H100
HBM3E
2024

HBM3E — 當前最高規格

在 HBM3 基礎上提升頻率至 9.6 Gbps，單顆最大容量達 36 GB（12-Hi），NVIDIA H200 與 GB200 搭載 8 顆可達 4.8 TB/s。AMD MI300X 更整合 8 顆達 5.3 TB/s，成為大型語言模型推理的頂配方案。

頻寬 ~1.2 TB/s / stack 堆疊 12-Hi / 16-Hi 容量 24–36 GB / stack 代表 GPU NVIDIA H200 · AMD MI300X
HBM4
2025+

HBM4 — 次世代（開發中）

預計 2025–2026 年量產，介面升至 2048-bit，單顆容量可能突破 48–64 GB，目標頻寬 2 TB/s / stack。SK Hynix 已宣佈量產路線圖，首發搭載於 NVIDIA Rubin 架構 GPU。

目標頻寬 ~2 TB/s / stack 介面 2048-bit 目標容量 48–64 GB / stack 代表 GPU NVIDIA Rubin（預計）

HBM 應用場景

超高頻寬與低功耗使 HBM 成為頻寬密集型工作負載的首選，但高成本也限制其主要應用於高端市場。

🤖

AI 模型訓練

大型 Transformer 訓練中，梯度與啟動值需頻繁讀寫，HBM 的超高頻寬可最大化 GPU 算力利用率，避免記憶體成為瓶頸。

💬

LLM 推理加速

生成 token 時需連續讀取數十億個模型權重參數，記憶體頻寬直接決定 token/s 輸出速度，HBM3E 讓單機推理速度提升 2–3 倍。

🔬

HPC 科學計算

氣候模擬、分子動力學、流體力學等計算密集型應用，大量矩陣運算高度依賴記憶體頻寬，HBM 是超級電腦節點的標配。

🎮

旗艦遊戲 GPU（早期）

HBM1 首次應用於 AMD Fury X，但成本過高難以普及消費端，後由 GDDR6/X 主導遊戲市場。HBM 現已退出消費級 GPU 市場。

🗄️

記憶體計算（PIM）

Processing In Memory 技術將簡單計算單元整合至 HBM 的 Logic Die，減少資料移動，進一步提升能源效率，為下一代架構方向。

📡

網路處理 / 智慧 NIC

高速封包處理（400G/800G）需要極低延遲的大容量查找表，HBM 整合於 ASIC 中可同時滿足頻寬與低延遲需求。

為何消費級顯卡不用 HBM？

成本因素：一顆 HBM3E 堆疊成本約 200–400 美元，8 顆即超過 2000 美元，遠超一張消費級 GPU 的總售價。

CoWoS 產能限制：矽中介層封裝需要特殊的 2.5D IC 整合製程（主要由台積電提供），產能受限、排單困難，無法支撐大規模消費市場出貨量。

GDDR7 已夠用：GeForce RTX 5090 採用 GDDR7，192-bit 介面仍可達 1.8 TB/s 頻寬，對大多數遊戲工作負載已綽綽有餘，無需 HBM 的極致頻寬。

供應鏈瓶頸：CoWoS 產能

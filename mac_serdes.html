MAC 晶片是什麼？

MAC（Media Access Controller，媒體存取控制器）是 OSI 第二層（資料鏈結層）的硬體實作，負責在 CPU/DMA 與 PHY 晶片之間處理以太網路幀。它是「封包的門衛」——確保格式正確、位址對齊、錯誤可偵測。

幀封裝與解封

MAC 負責在發送時加上目的 MAC 位址、來源 MAC 位址、EtherType/長度欄位，以及尾端的 FCS（CRC-32）。接收時則做相反動作，並驗證 CRC 是否正確。

流量控制（Flow Control）

全雙工模式下，MAC 支援 IEEE 802.3x PAUSE 幀，以及更精細的 PFC（Priority Flow Control）來避免接收緩衝區溢位。對資料中心交換機至關重要。

半雙工 CSMA/CD（歷史）

早期乙太網路 MAC 需要偵聽載波（Carrier Sense）並處理碰撞（Collision Detection）。現代交換式全雙工網路已不再需要，但 MAC 名稱沿用至今。

與 CPU 的介面：DMA

MAC 透過 DMA（Direct Memory Access）將封包直接在記憶體與 MAC 之間搬運，不需 CPU 介入每個位元組。驅動程式管理 TX/RX 描述符環（Descriptor Ring）。

MAC 的存在形式
  • 獨立 MAC 晶片：如 Microchip LAN9303，常見於嵌入式系統搭配外部 PHY
  • 整合於 SoC：手機、路由器、嵌入式 CPU 通常內建 MAC
  • 整合於交換機 ASIC：Broadcom Tomahawk/Tofino 等含數百個 MAC port
  • 整合於 NIC：Intel、Mellanox 等網卡晶片 MAC＋PHY 合一
MAC 不做什麼
  • 不處理 IP 位址（那是 L3 路由器的工作）
  • 不做電氣訊號轉換（那是 PHY 和 SerDes 的工作）
  • 不了解 TCP/UDP（那是 L4 以上）
  • 現代 NIC 可卸載 TCP checksum、TSO 等，但 MAC 核心仍只管 L2

PHY 晶片是什麼？

PHY（Physical Layer Transceiver，實體層收發器）是介於 MAC 和傳輸介質之間的橋樑晶片。它的內部包含了 PCS、PMA，以及最底層的 SerDes 電路，是「把數位幀翻譯成電氣/光訊號」的翻譯官。

PCS（Physical Coding Sublayer）

負責對數位資料進行編碼，常見有 8B/10B（GbE）、64B/66B（10GbE 以上）等。編碼確保足夠的位元轉換頻率，方便接收端的 CDR 鎖相，並提供 DC 平衡。

PMA（Physical Medium Attachment）

PMA 管理 SerDes lane 的對齊（Alignment）、通道綁定（Lane Bonding）、極性翻轉等。對於多 lane 高速介面（如 CAUI-4），確保 4 條 lane 到達時對齊。

PMD（Physical Medium Dependent）

PMD 是與傳輸介質直接接觸的部分：銅線 PHY 處理 PAM5/PAM4 基帶訊號；光纖 PHY（或光模組）驅動雷射二極體。

SerDes 在 PHY 裡的位置

SerDes（Serializer / Deserializer）就嵌在 PHY 的 PMA 子層內部。它做最原始的並串轉換，並產生高速差動電流驅動訊號。PHY = PCS + PMA + SerDes + PMD。

Auto-Negotiation（自動協商）

PHY 負責執行 IEEE 802.3 Auto-Negotiation，自動偵測對端速率（10M/100M/1G/10G）與雙工模式，並告知 MAC 以便調整。

MDIO 管理介面

MAC 透過 MDIO（Management Data Input/Output，IEEE 802.3 Clause 22/45）這條雙線匯流排設定並讀取 PHY 暫存器：連線狀態、速率、錯誤計數、均衡器調整等。

🔌

SerDes 在訊號鏈中的確切位置

SerDes 是最靠近「銅線/光纖」的類比混合訊號電路。它既不理解以太網路幀，也不知道 IP 位址——它只管把 N-bit 並列資料轉成一條超高速串列差動訊號，以及反向還原。

SerDes 的輸入：來自 MAC

MAC 透過寬並列匯流排（如 XGMII = 32-bit @ 312.5 MHz × 2 DDR = 10 Gbps）把資料傳給 PHY，PHY 的 PCS 做完編碼後，餵給 SerDes 的 TX FIFO（通常 20~40 bit 寬）。

SerDes TX：並串轉換

PLL 將參考時脈倍頻，Serializer 以極高速（10G / 25G / 56G / 112G bps）逐 bit 輸出到差動 TX 對（差動電壓 ±200~800 mV）。TX FIR 均衡器修正預加重。

SerDes RX：串並轉換

CTLE 補償通道損耗，DFE 消除符號間干擾，CDR 從訊號中恢復時脈並對齊取樣點，Deserializer 把串列位元還原為並列資料輸出給 PCS。

SerDes 的輸出：到傳輸介質

差動對直接驅動：銅纜（SFP+ Direct Attach、CAT-8 網路線）、或光模組內的 EML/VCSEL 雷射二極體（光纖）。100G 以上通常為多 lane，每 lane 一個 SerDes。

MAC ↔ PHY 之間的並列介面（XGMII 範例：10G）
MAC XGMII TX 32-bit 資料 + 4-bit 控制 × 312.5 MHz DDR = 10 Gbps → PHY PCS 64B/66B 編碼 → SerDes TX 串列 10.3125 Gbps NRZ → 差動對

常見問題：「SerDes 和 PHY 是同一個嗎？」

不完全是。SerDes 是一個電路元素（並串轉換器＋PLL/CDR），PHY 晶片是一個完整晶片產品，內部包含 SerDes 作為其中一個子電路。 就像「引擎」和「汽車」的關係：SerDes 是引擎，PHY 是整台車（還有變速箱 PCS、油門 MDIO、車燈 Auto-Negotiation）。 有些 SoC 把 SerDes 從 PHY 中獨立出來當作通用 I/O，例如 FPGA 的 GTX/GTH transceiver，也可以直接用作 PCIe、SATA、HDMI 等。

USB HUB 內部架構

🔗 USB 拓撲限制

  • USB 規格最多允許 5 層 HUB 串接（USB 2.0：最深 7 層）
  • 每個 HUB 最多 7 個下游端口（USB 2.0 Spec §11.23）
  • 整個 USB 樹狀結構最多 127 個裝置（7-bit 位址）

⚡ 電源架構

  • Bus-Powered：從 Host 取電，受限於 500mA/900mA
  • Self-Powered：外接 DC，每端口可獨立提供 900mA
  • BC 1.2 充電端口可達 1.5A，USB-PD 最高 5A@20V

📊USB 2.0 HUB：共享匯流排頻寬示意

USB 2.0 HUB 所有下游端口共享同一條 480 Mbps 上行頻道。越多裝置同時傳輸，每個裝置實際頻寬越少。

＊理論值，實際因 USB 協定 overhead（token packet、handshake 等）有效吞吐約為標稱值的 60~80%
⚡USB 3.x HUB：SuperSpeed 各端口獨立頻寬

💡 關鍵差異：Shared vs. Per-Port

USB 2.0 HUB 使用「Hub Repeater」廣播機制，所有端口共享 480 Mbps；而 USB 3.x HUB 對 SuperSpeed 通道使用「Router」架構，下游端口到 HUB IC 之間各自有獨立 5/10 Gbps 通道。但整個 HUB 對 Host 只有一條上行鏈路，因此 4 個裝置全力傳輸時仍需分享上行頻寬。

Transaction Translator（TT）原理

🔄為什麼需要 TT？

⚠️ 速度不相容問題

USB 2.0 HUB 工作在 HS（480 Mbps），但鍵盤、滑鼠等老裝置是 FS（12 Mbps）或 LS（1.5 Mbps）。如果直接連到 HS 匯流排，速度不匹配會導致整條匯流排降速至 FS 或 LS——所有高速裝置都被拖慢。

✅ TT 的解法

Transaction Translator 在 HUB 內部扮演「速度橋接器」：在 HS 幀中收到 Host 的 Split Transaction（START SPLIT），先暫存資料，再用 FS/LS 速度與低速裝置溝通，完成後在後續 micro-frame 回傳結果（COMPLETE SPLIT）。HS 匯流排只看到 Split 封包，不降速。

⚖️Single TT vs. Multi-TT

Single TT（共用）

  • 成本低、晶片面積小
  • 多個 FS/LS 裝置同時使用時互搶 TT，延遲增加
  • 適合低速裝置少（鍵盤+滑鼠）的情境

Multi-TT（每端口獨立）

  • 每端口有獨立 TT 佇列
  • 多個 FS/LS 裝置可平行處理
  • 適合工業設備、多週邊應用

🛠ODM/OEM 廠商開發流程
1

IC 選型與 Datasheet 研讀

依據端口數、速度（USB 2.0/3.x）、電源方式（Bus/Self）、成本預算選擇 IC。向 Genesys / VIA Labs 申請完整 Datasheet（需 NDA）。

2

原理圖設計（Schematic）

參考 IC 廠商提供的 Reference Design，設計電源（LDO、DC-DC）、每端口 Power Switch IC（SY6280、RT9711）、ESD 保護、EEPROM、晶振電路。

3

PCB 佈線（Layout）

USB 差動訊號需受控阻抗走線（90Ω for USB 2.0，85Ω for USB 3.x）、等長配對、減少 via、USB 3.x SuperSpeed 需要嚴格的 TX/RX 對稱走線。

4

EEPROM 配置（選配）

使用 IC 廠商提供的 GUI 工具燒錄 EEPROM，設定 VID/PID（需向 USB-IF 申請）、產品名稱字串、端口電流配置、BC1.2 模式等。

5

USB-IF 認證 & EMC 測試

通過 USB-IF Compliance Test（USBCV、USBET）取得 USB-IF 認證標誌。進行 EMC（CE/FCC）測試，確保輻射不超標。

🔧 USB 3.x 韌體開發工具

  • GL3523 SDK：Genesys 提供 ARM GCC 工具鏈 + 範例碼，開發者修改 HUB Class 擴充功能
  • SMBus 配置：Microchip USB5734 支援 SMBus 動態配置，可由主機 MCU 在運行時更改端口設定
  • OTP 燒錄：部分 IC 支援一次性可程式化 OTP 取代 EEPROM，量產更省 BOM

🐧 Linux Kernel HUB 驅動架構

  • drivers/usb/core/hub.c — 核心 HUB 驅動，處理枚舉、端口事件
  • usb_hub_probe() — HUB 被識別後初始化所有下游端口
  • hub_port_connect() — 偵測裝置插入，啟動 Reset 序列
  • Hub Class Request（GET_STATUS, SET_FEATURE 等）透過 USB 控制傳輸發送給 HUB IC

🐧Linux 中的 USB HUB：有沒有 Linux 跑在 HUB 裡？

✅ 一般情況：HUB IC 本身不跑 Linux

消費性與工業標準 USB HUB IC 內部是硬體狀態機或小型 MCU（8051/ARM-M0），執行的是 bare-metal 韌體，不跑 Linux。Linux 跑在連接 HUB 的 Host 端（PC、SBC 如 Raspberry Pi）。

🔮 特殊案例：智慧型 HUB（Managed Hub）

部分企業級「Managed USB Hub」（如 Acroname、Cypress EZ-USB HX3）內建較強 MCU，可透過 USB 或 Ethernet 遠端控制每個端口的電源與連線，有些甚至跑 RTOS（FreeRTOS）但不是完整 Linux。

🤖 Raspberry Pi / SBC 上的 USB HUB

Raspberry Pi 4 內建 VL805 USB 3.0 Host Controller + 一個 4-port HUB，由板上 ARM Cortex-A 跑 Linux 管理。這裡 Linux 是跑在「使用 HUB」的 SoC 上，而非 HUB IC 本身。在工業應用中，有人用 ESP32、STM32 搭配外部 USB HUB IC（如 USB2514B 透過 SMBus 配置），MCU 管理 HUB 的電源開關，但此 MCU 跑 FreeRTOS 而非 Linux。

💡常見 QA
Q：USB HUB 為什麼可以一個 IC 做到多個 Port？

USB 協定是「主從架構」，Host 主導所有通訊。HUB 內部的 Hub Repeater（USB 2.0）或 SS Router（USB 3.x）在 IC 內部以硬體邏輯實作。每個端口的物理 PHY（差動收發器）由 IC 內建的多路 PHY 陣列提供，IC 製造商在設計時就把 N 個 PHY 整合進去，透過內部匯流排與 Hub Repeater 連結。本質上是「IC 的面積換端口數」。

Q：USB HUB 插越多裝置速度越慢嗎？

USB 2.0 HUB：是的，共享 480 Mbps，同時傳輸時各裝置平分頻寬。USB 3.x HUB：SuperSpeed 通道每端口各自連到 HUB IC，不互相競爭；但所有端口仍共用一條上行 SuperSpeed 到 Host，所以 Host 端的上行仍是瓶頸。例如 4 個裝置接 USB 3.2 Gen1 HUB，上行總頻寬 5 Gbps 由 4 個裝置共享。

Q：USB HUB 在 Linux 如何被識別？

插上後 lsusb 可見 HUB 裝置（bDeviceClass=09 Hub），lsusb -t 顯示樹狀結構。dmesg 可見 usb X-Y: new high-speed USB device number Z 訊息，接著 hub.c 啟動端口偵測。可用 cat /sys/bus/usb/devices/X-Y/bNbrPorts 查詢端口數、bTTThinkTime 查 TT 設定。

Q：可以自製 USB HUB 嗎（自己設計電路）？

完全可以！購買 GL850G 或 FE1.1s（淘寶/立創有售），參考 Datasheet Reference Design 畫原理圖，加上 12MHz 晶振、SY6280 Power Switch、ESD 保護、Type-A 座，即可做出 4-port USB 2.0 HUB。不需要任何韌體，上電即可使用。KiCad/Altium 繪製 PCB 後，可至 JLCPCB 打樣約 50 元人民幣。

Makefile 賦值語法：
=、:=、?=、+= 到底差在哪？

先別把它當成四種「等號」。它們是在決定：何時展開變數、能否被覆寫、以及是否接在原值後面。

一眼看懂四種運算子展開時機對照表選擇器實作模式

一眼看懂：變數像「食譜卡」或「已備好的食材」

Make 會在讀取 Makefile 與執行 recipe 的不同時間點處理文字。關鍵不是值長什麼樣，而是它何時被取用。

=先記下食譜
用到才展開
:=現在就備料
立刻固定
?=沒有才填入
提供預設值
+=在原食譜後
繼續追加
先懂這個：$(VAR) 是取用變數的寫法；$(shell date)、$(wildcard *.c) 這類函式何時被展開，會直接改變結果與效能。

四位角色與各自的任務

= 遞迴展開（recursive）

把右側原封成「規則」保存；每次取用時才展開。適合希望後續變數變動也能反映的情境。

BASE = hello
MSG = $(BASE) world
BASE = hi
# $(MSG) → hi world

:= 立即展開（simple）

讀到這行立刻計算右側，結果固定。適合昂貴函式或你要明確保留當下值的情境。

BASE = hello
MSG := $(BASE) world
BASE = hi
# $(MSG) → hello world

?= 條件預設值

只有變數「尚未定義」才賦值；命令列或環境先給的值會保留。

CC ?= gcc
# make CC=clang 不會覆蓋 clang

+= 追加內容

接在既有值後方；會保留原變數的展開特性。常用來累積 flags、來源檔或路徑。

CFLAGS := -Wall
CFLAGS += -O2
# -Wall -O2

流程圖：同一份字串，兩種不同的取值時機

1. 讀檔Make 由上到下讀取賦值。
2. 遇到 =保存右側文字，暫不完全計算。
3. 遇到 :=立刻展開右側並保存結果。
4. 用到變數= 此刻才計算；:= 直接取固定值。
常見誤解：= 不是「普通等號」、:= 不是「更強的等號」。兩者最重要的差異是展開時間。若右側有 $(shell ...)，使用 = 可能讓命令在每次取用時重跑。

快速對照：選錯時通常會發生什麼？

語法何時展開右側覆寫規則典型用途
VAR = value每次使用 VAR 時後面的賦值可改變依賴結果依賴其他會變的變數
VAR := value讀到這一行時保留當下計算值快取 wildcard / shell 結果
VAR ?= value只有未定義才設定尊重命令列、環境或先前設定可調整的預設 compiler / flags
VAR += value依 VAR 原本風格追加，不是取代累積 CFLAGS、SRCS

可直接帶走的 Makefile 模式

預設編譯器與可追加旗標

CC ?= gcc
CFLAGS := -Wall -Wextra
CFLAGS += -O2

app: main.c
	$(CC) $(CFLAGS) $< -o $@

使用者可以 make CC=clang；專案仍保有基本警告與最佳化。

收集來源檔：算一次就好

SRC := $(wildcard src/*.c)
OBJ := $(SRC:.c=.o)

app: $(OBJ)
	$(CC) $^ -o $@

用 := 將檔案清單固定在 Make 讀檔階段，閱讀與除錯較直覺。

你可以這樣想：先問「這個值需要晚點才知道嗎？」需要，就考慮 =；不需要而且想穩定、少算幾次，就用 :=。接著再問是否為預設（?=）或追加（+=）。

新手練習順序

  1. 先在最小 Makefile 中改動 BASE，觀察 = 與 := 的輸出。
  2. 把 CC ?= gcc 以 make CC=clang 執行一次。
  3. 用 += 增加 -g，確認原有旗標沒有消失。
  4. 最後才把 wildcard、pattern rule 與自動變數 $@、$< 串起來。

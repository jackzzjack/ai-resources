什麼是 Graphify？

Graphify 是一個專為 AI 程式碼助手（如 Claude Code）設計的知識圖譜工具。它會分析你的整個專案目錄（包含程式碼、文件、SQL Schema、甚至圖片與影片），將其壓縮成一個結構化的知識圖譜 (Knowledge Graph)。

💡 為什麼需要它？ 傳統 AI 助手通常依賴 grep 或全檔讀取來理解專案，當專案變大時，每次詢問都會消耗大量 Token，且難以理解跨檔案的結構關係。Graphify 預先完成繁重的分析，讓 AI 透過查詢圖譜來回答問題。
💰 節省 70%+ Token 透過圖譜查詢取代重複讀取原始檔案，顯著降低 API 使用成本。
⚡ 極速上下文 AI 能在幾秒內理解萬行代碼的架構，無需逐一掃描檔案。
🕸️ 結構化理解 理解函數調用、類別繼承與數據流向，而非僅僅是關鍵字。

安裝步驟

Graphify 推薦使用 uv 或 pipx 進行安裝，以確保 CLI 工具在全域可用。

⚠️ 注意：PyPI 名稱 官方套件名稱為 graphifyy (字尾有兩個 y)，請勿裝錯。

1. 使用 uv 安裝（推薦）：

uv tool install graphifyy

2. 或使用 pipx 安裝：

pipx install graphifyy

3. 註冊 AI 助手技能（以 Claude Code 為例）：

graphify install

這會在你的 AI 助手環境中註冊 /graphify 指令。

如何在專案中使用？

安裝完成後，你可以在支援的 AI 助手（如 Claude Code, Codex）中使用以下流程：

第一步：初始化圖譜 在專案根目錄下，詢問 AI：
「/graphify」
AI 會開始掃描專案並建立 graphify-out/ 資料夾。

常用情境範例：

  • 架構導覽：「請根據圖譜解釋這個專案的 Auth Flow 是如何實現的？」
  • 影響分析：「如果我修改了 user.ts 中的 login 函數，會影響到哪些組件？」
  • 視覺化： 執行 graphify export callflow-html 生成可互動的調用鏈網頁。

你也可以直接在終端機開啟視覺化圖譜：

# 在瀏覽器打開圖譜網頁
open graphify-out/graph.html

核心功能清單

🔍 多模態索引 除了程式碼，還支援 SQL、PDF、甚至是錄製的設計會議影片（需 AI 模型支援）。
📊 可視化報告 自動生成 GRAPH_REPORT.md，摘要專案的核心概念與驚人關聯。
🗺️ Call Flow Export 將複雜的函數調用鏈導出為 Mermaid 圖表或獨立 HTML 頁面。
🔒 隱私優先 圖譜數據存儲在本地磁碟，不會在未經授權的情況下上傳原始代碼。

5G 核心網路元件說明

AMF
Access and Mobility Management Function
負責 UE 的接入管理與移動性管理，終結 NAS 訊令（N1），並協調基地台（N2）。處理 UE 的註冊、認證流程發起、位置更新、尋呼。
SMF
Session Management Function
管理 PDU Session 的建立、修改與釋放。分配 IP 位址、設定 QoS、透過 N4 控制 UPF 的封包轉發規則（PFCP），並向 PCF 取得策略。
UPF
User Plane Function
用戶平面核心元件，負責封包路由與轉發（GTP-U）、QoS 執行、流量偵測（Traffic Detection）。連接 RAN（N3）、其他 UPF（N9）與資料網路（N6）。
PCF
Policy Control Function
統一策略框架，提供 AM Policy（存取移動策略）與 SM Policy（Session 策略、PCC rules）。整合 UDR 中的訂閱策略資料。
UDM
Unified Data Management
管理用戶訂閱資料的統一前端，包含用戶識別碼轉換（SUPI/SUCI）、訂閱資料存取、認證憑據管理，底層資料儲存於 UDR。
AUSF
Authentication Server Function
執行 5G 認證流程（5G-AKA / EAP-AKA'）。向 UDM 取得認證向量，協助 AMF 完成 UE 身份驗證與安全密鑰的產生。
NRF
Network Repository Function
5G 的「服務目錄」。所有 NF 啟動後向 NRF 註冊自身 Profile（能力、位址等），其他 NF 透過 NRF 探索（Discovery）並訂閱服務。
NSSF
Network Slice Selection Function
根據 UE 請求的 NSSAI 及運營商策略，選擇合適的 Network Slice。返回 Allowed NSSAI 與對應服務該 Slice 的 AMF 集合。
NEF
Network Exposure Function
5G 對外的 API 閘道器。安全地將 5G Core 能力（QoS 控制、UE 位置、網路狀態）暴露給第三方 AF，並保護核心網路安全。
UDR
Unified Data Repository
統一資料儲存庫，儲存訂閱資料、策略資料、應用資料。為 UDM、PCF、NEF 提供資料存取服務（Nudr 介面）。
CHF
Charging Function
5G 統一計費功能，支援線上計費（OCS）與離線計費（OFCS）。SMF 透過 N40 介面回報用量，CHF 進行扣費或記錄。
LMF
Location Management Function
5G 定位功能，透過 NR Positioning（NR-OTDOA、UL-TDOA、E-CID 等）提供高精度定位服務，支援緊急呼叫位置回報。
gNB
Next Generation NodeB（5G基地台）
5G NR 基地台，支援 Sub-6GHz 與 mmWave。內部可拆分為 gNB-CU（集中單元）與 gNB-DU（分散單元），透過 F1 介面連接。支援 Massive MIMO、Beamforming。
AF
Application Function
第三方應用服務，可透過 NEF（非信任）或直接透過 SBI（受信任 AF）與 5GC 互動，申請 QoS 保障、獲取網路資訊、訂閱網路事件。
ng-eNB
Next-Generation eNodeB（4G 升級版）
在 NSA（非獨立組網）模式下，4G eNB 透過升級連接至 5G Core，與 gNB 形成 Dual Connectivity（DC），加速 5G 初期部署。
UE
User Equipment（用戶設備）
5G 終端設備，包含智慧手機、IoT 感測器、工業設備、車載 V2X 模組等。內含 USIM 存放 SUPI（IMSI），支援 5G-AKA 認證。

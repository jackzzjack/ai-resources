①
什麼是 SerDes

SerDes（Serializer / Deserializer）是一種混合訊號電路，負責在並列資料（Parallel Data）與串列資料（Serial Data）之間進行轉換。透過將多條低速並列資料合併成單條（或少數條）高速串列訊號，可大幅減少 PCB 走線數量、降低成本並提升傳輸速率。它是現代高速介面（PCIe、以太網路、USB、SATA、DisplayPort 等）不可或缺的核心 PHY 電路。

Serializer（序列化器）

將 N 位元並列資料轉換成 1 條（或少數條）高速串列訊號。通常包含並列轉串列（P2S）移位暫存器、Mux，以及高頻時脈驅動邏輯。

Deserializer（解序列化器）

從高速串列訊號中還原出 N 位元並列資料。需要先進行時脈資料回復（CDR），再透過串列轉並列（S2P）電路輸出。

為什麼需要 SerDes
  • 並列介面受 Skew 限制，無法超過幾 GHz
  • 串列高速線對更少，EMI 干擾降低
  • 差動訊號（Differential）抗雜訊佳
  • 單通道速率持續突破（1.6 Tbps/lane 達成中）
常見應用場景
  • PCIe Gen5/6（32 / 64 GT/s）
  • 400G / 800G Ethernet（SERDES 112G、224G）
  • HBM、HBM3E 記憶體介面
  • USB 4 / Thunderbolt 4/5
  • DisplayPort 2.1、HDMI 2.1

調變方式：NRZ vs PAM4

SerDes 的調變方式決定了每個符號（Symbol）能攜帶的資訊量。主流選擇為 NRZ（2 電位）與 PAM4（4 電位），各有優缺點。

NRZ（Non-Return-to-Zero）

每個符號代表 1 bit（0 或 1）。電位擺幅大，雜訊容限高，實作較簡單。適用於 ≤ 28 Gbps 場景。隨速率提高，通道損耗急劇增加，超過 56 Gbps 後難以使用。

  • 眼圖：2-level（1 個眼孔）
  • 雜訊容限：SNR 需求低
  • 等化難度：相對容易

PAM4（Pulse Amplitude Modulation 4）

每個符號代表 2 bit（4 個電位：00、01、10、11）。同樣的符號率下傳輸量是 NRZ 的 2 倍，但眼圖高度縮小為 NRZ 的 1/3，對雜訊容限、FEC、等化要求大幅提高。

  • 眼圖：4-level（3 個眼孔）
  • 雜訊容限：比 NRZ 低約 9.5 dB
  • 必須搭配 RS-FEC（前向糾錯）
⚠️
PAM4 的 3 個眼孔高度僅 NRZ 的 1/3，加上等化殘差與串擾，實務上 BER 遠差於 NRZ。因此 IEEE 802.3 400G/800G 規格皆強制要求 RS-FEC（Reed-Solomon Forward Error Correction）來補足錯誤率。
⑤
等化器（Equalizer）

通道（PCB 走線、連接器、Cable）會造成高頻損耗，使訊號眼圖閉合。等化器的任務是補償這些損耗，重新打開眼圖。SerDes 通常同時配備 TX EQ 與多級 RX EQ。

TX FIR（Pre-/De-emphasis）

在發送端提前對高頻分量補強（Pre-emphasis）或對低頻分量衰減（De-emphasis），以對抗通道高頻損耗。常見為 3-tap（pre, main, post）或更多 tap 的 FIR 濾波器。

  • Coefficient: C(-1)、C(0)、C(+1)
  • PCIe 規格定義 Preset 0–10
  • 透過 Link Training 自動協商
CTLE（連續時間線性等化）

RX 端的主動 RC 濾波器，對高頻進行提升（peaking）。電路形式為帶有高頻 zero 的差動放大器。可程式化 DC gain 與 peaking 頻率，由 adaptation 演算法控制。

  • 補償通道的 −20 dB/decade 損耗特性
  • 過度 peaking 會放大高頻雜訊
  • 與 DFE 配合使用效果最佳
DFE（Decision Feedback Equalizer）

非線性等化器，利用已判決的符號回饋來消除符號間干擾（ISI）。對於後游標 ISI（post-cursor）特別有效，且不放大雜訊（優於 CTLE 的關鍵優點）。

  • H1～H5 tap（或更多）回饋係數
  • 限制：不處理前游標 ISI（需 TX EQ）
  • 高速下 critical timing path（需 unrolling）
FFE（Feed-Forward Equalizer）

純線性前饋等化器，可同時處理前游標與後游標 ISI，常用於 PAM4 高速設計（112G+）。與 CTLE 相比，可程式化彈性更高。

  • Rx FFE 為 7~25 tap FIR
  • 搭配 LMS 演算法自動適應
  • 消耗功率較高
⑥
時脈資料回復（CDR）

串列傳輸不傳送時脈訊號，RX 端必須從接收到的資料中自行重建時脈，這就是 CDR 的任務。CDR 通常以 PLL 或 DLL 為基礎實作。

Linear CDR（PLL-based）

以 PLL 為核心，相位偵測器（PD）比較輸入資料邊緣與本地時脈，誤差訊號經低通濾波後控制 VCO 頻率，逐步鎖定至資料速率。

  • 帶寬決定 Jitter Tracking 範圍
  • VCO 相位雜訊直接影響 RX 抖動
  • Mueller–Müller PD 或 Alexander PD
Bang-Bang CDR

相位偵測器只輸出 +1/−1（非線性），結構簡單、速度快，適合高速設計。缺點是存在 Limit Cycle（有界振盪），引入確定性抖動（DJ）。

  • 廣泛用於 10G+ NRZ SerDes
  • Loop filter 設計關鍵：避免 LC 過大或過小
  • Jitter 容限需符合規格（UI 內）
參考時脈架構

現代 SerDes 通常採用外部參考時脈（RefClk）搭配 CDR 的雙迴路架構：

  • SATA/PCIe SSC：RefClk 允許展頻（Spread Spectrum Clocking），CDR 需追蹤
  • Clock Forwarding：另外傳送時脈差動對（如 MIPI C-PHY）
  • Embedded Clock：從資料中嵌入時脈資訊（常見於光纖）
PAM4 CDR 特殊挑戰

PAM4 的眼圖只有 3 個較小的眼孔，相位偵測更困難。通常採用 Multi-level PD 並搭配更強的 DSP 才能可靠鎖定。

  • Phase detector 需支援 4 電位判決
  • Adaptation 演算法更複雜
  • FEC 保護 CDR 初始鎖定階段
⑦
編碼方案

編碼在資料送入 SerDes 前將其轉換，以確保足夠的訊號轉換（transition）頻率（有助 CDR 鎖定）及直流平衡（DC balance），並提供控制字元（K-code）機制。

編碼方式 效率 特色 使用場景
8b/10b 80% DC 平衡、Running Disparity、豐富 K-code；硬體查表實作簡單 PCIe Gen1/2、SATA、USB 3.0、Fibre Channel
64b/66b 97% 2-bit Sync Header 取代大量 overhead；用 Scrambler 確保 DC 平衡 PCIe Gen3/4/5、10G/25G/100G Ethernet
128b/130b 98.5% 效率更高；搭配 Scrambler；PCIe Gen6 改為 FLIT 模式（256B 固定封包） PCIe Gen6、USB 3.2、DisplayPort 2.1
RS-FEC（加法） ~91%（KP4） 並非傳統行編碼，而是前向糾錯碼；KR4 / KP4 為主流；PAM4 必備 400G / 800G Ethernet（PAM4）
PRBS 加擾（Scrambler） 100%（無額外開銷） 將資料與 PRBS 序列 XOR，使頻譜分散；不是糾錯，只是打散長串連 0/1 配合 64b/66b / 128b/130b 使用
ℹ️
為什麼需要 DC 平衡？若訊號長期偏向高電位或低電位，交流耦合電容（AC coupling cap）兩端電壓差會影響判決閾值，導致誤碼。8b/10b 的 Running Disparity 機制，以及 64b/66b 的 Scrambler，都是解決此問題的方法。
⑧
訊號完整性（Signal Integrity）

訊號完整性問題是高速 SerDes 設計的最大挑戰之一。下方涵蓋主要的 SI 效應與應對策略。

插入損耗（Insertion Loss）

PCB 介電材料的介電損耗（Dielectric Loss）與導體的趨膚效應（Skin Effect）造成高頻訊號衰減。在 28 GHz 下，普通 FR-4 每 inch 損耗約 1.5–2 dB。

  • 解法：使用低損耗基板（Megtron 6、Rogers）
  • 縮短走線長度，減少過孔（Via）
  • 增加 TX pre-emphasis + RX CTLE/DFE
反射（Reflection）

阻抗不連續點（過孔、連接器轉接、佈線寬度突變）造成訊號反射。反射訊號疊加在主訊號上，形成 ISI 並縮小眼圖。

  • 差動阻抗維持 85–100 Ω（典型 85 Ω）
  • 避免走線 Stub（過孔反鑽 Back-drill）
  • 連接器選型：確認 Return Loss 規格
串擾（Crosstalk, NEXT/FEXT）

相鄰差動對之間的電磁耦合造成干擾，分為 Near-End Crosstalk（NEXT）與 Far-End Crosstalk（FEXT）。PAM4 因眼圖較小，對串擾更敏感。

  • 差動對間距 ≥ 3W（W = 走線寬度）
  • 相鄰通道盡量反向走線（引入混合串擾抵消）
  • 使用 XTALK-aware 3D EM 仿真驗證
抖動（Jitter）

抖動指時脈邊緣偏離理想位置的時間偏差，是 SerDes 可靠性的核心指標。可分為確定性抖動（DJ）與隨機抖動（RJ）。

  • TJ（Total Jitter）= DJ + n × RJ
  • 典型規格：TJ < 0.3 UI（PCIe Gen5）
  • 量測工具：抖動分析儀、BERT、示波器
電源完整性（Power Integrity）

SerDes 類比電路（PLL VCO、Driver）對電源雜訊極敏感。電源軌上的 SSN（Simultaneous Switching Noise）和 PDN 阻抗峰值都會轉換為相位雜訊。

  • 獨立供電軌（AVDD 與 DVDD 分離）
  • 多級去耦電容（板端 + 封裝端 + Die 上 MIM cap）
  • PDN 阻抗仿真：目標平坦 < 10 mΩ
SSC（展頻時脈）與 Jitter Tolerance

PCIe/SATA 要求支援 SSC（Spread Spectrum Clocking，±0.5% 三角調製），以降低 EMI 輻射。CDR 必須有足夠帶寬追蹤 SSC，同時不放大高頻抖動。

  • CDR 帶寬 vs. Jitter Tolerance 的 trade-off
  • SSC Down-spread vs. Center-spread
  • 需對 SSC profile 做 Jitter Tolerance 測試
⑨
實作注意事項

SerDes 設計橫跨電路設計、PCB layout、韌體 / 驅動程式多個層面。以下整理各層面的關鍵注意事項。

PCB Layout

差動對佈線原則
  • 兩線同層、同寬、同長（Match length < 5 mil）
  • 差動對間距：1W～1.5W（緊耦合）
  • 避免在差動對中間放置過孔或測試點
  • 換層時 P/N 兩線同時換層，加 GND 過孔回流
過孔（Via）處理
  • 高速差動對過孔建議 Back-drill 去除 Stub
  • 每個訊號過孔旁需配置 GND 回流過孔（< 20 mil）
  • 避免多余的測試點電容效應（> 25 GHz 顯著）
  • 使用 3D EM 工具（HFSS / CST）仿真過孔 S 參數
參考層連續性
  • 差動走線下方 GND/PWR 參考層不得有分割縫隙
  • 跨電源域時需在分割縫隙旁加橋接電容
  • GND 平面完整性比訊號走線更重要
AC 耦合電容
  • 一般使用 0.1 µF（低頻截止 < 1 kHz）
  • 電容 Pad 兩端需阻抗匹配設計（避免凸起）
  • PAM4 高速：注意電容諧振頻率（SRF）
  • PCIe 規範明確規定 AC 耦合位置（TX 端）

類比電路設計

PLL / VCO 設計
  • VCO tuning range 需涵蓋 PVT 變化（±10–20%）
  • LC-VCO 優於 Ring VCO（相位雜訊）
  • Loop filter 電容佔用面積大，考慮外置選項
  • PLL 帶寬：通常為 參考時脈頻率的 1/10
CTLE 設計
  • Peaking 頻率應略低於奈奎斯特頻率
  • DC gain & AC gain 獨立可調（3–4 bit 精度）
  • Adaptation 演算法需對不同通道收斂
  • CTLE → DFE 的 gain partitioning 需仔細分配
DFE Unrolling
  • 1st tap DFE 在超高速（>28 Gbps）下 critical path 嚴苛
  • 需採用 Speculative DFE（Loop Unrolling）技術
  • 代價：功耗與面積增加約 2×
  • 2nd tap 以上通常不需 unrolling
TX Driver 設計
  • 電流模式（CML）Driver：高速首選，但功耗較高
  • 電壓模式 Driver：功耗低，但驅動能力受限
  • Pre-driver 的 slew rate 控制避免非線性失真
  • PAM4 Driver：4 電位精度要求 INL/DNL < 0.5 LSB

韌體 / 驅動程式

Link Training 流程
  • PCIe LTSSM（Link Training and Status State Machine）需完整實作
  • Polling → Detect → Config → L0 各狀態轉換條件
  • Equalization Phase 0/1/2/3（Gen3+ 需完整協商）
  • 逾時處理與 retry 機制不可省略
Adaptation 演算法
  • LMS（Least Mean Squares）：最常見，但需調 step size
  • Zero-Forcing：對 ISI 最佳化，但對雜訊敏感
  • MMSE（Minimum Mean Square Error）：平衡 ISI 與雜訊
  • Convergence 速度 vs. 穩態誤差的 trade-off
眼圖監測與 BER 估算
  • 嵌入式眼圖掃描（Eye Scan）用於 debug
  • FEC 計數器：監控 Symbol Error Rate 趨勢
  • BER Floor 偵測：若 FEC 修正量持續增加需告警
  • 定期跑 Link Margin Test（垂直/水平眼圖餘量）
電源管理（ASPM）
  • PCIe ASPM L0s / L1 / L1.1 / L1.2 各子狀態
  • 進入低功耗狀態時 SerDes PHY 需正確關閉 PLL
  • 喚醒序列（P1 → P0）需符合規格時序
  • 測試：ASPM 連續切換下 BER 是否惡化
🔴
常見設計陷阱：忽略 PVT（製程 / 電壓 / 溫度）變化是高速 SerDes 失敗最常見的原因。等化器係數在 −40°C 與 +125°C 下可能相差 2× 以上，必須在設計初期就建立完整的 PVT 角落仿真計畫（Corner Simulation）。
✅
驗證流程建議：採用 IBIS-AMI 模型進行通道仿真（Keysight ADS / Mentor HyperLynx），搭配實際 BERT 量測。先確認 S 參數符合規格，再進行抖動分解分析（TJ/DJ/RJ 拆解），最後做壓力測試（溫度 / 電壓邊界）。
⑩
實作清單

完成 SerDes 設計與量產前，建議逐項確認以下事項。

SI
通道 S 參數仿真使用 3D EM 工具完成 PCB 走線、過孔、連接器的 S 參數提取，確認 IL/RL/NEXT/FEXT 符合規格。
SI
IBIS-AMI 通道仿真以 AMI 模型跑 10,000+ bit 統計眼圖，確認 BER < 10⁻¹²（或含 FEC 後的目標）。
PCB
差動對長度匹配P/N 兩線匹配 < 5 mil；差動對間 skew < 1 ps。
PCB
Back-drill 驗收確認過孔 stub 長度 < 10 mil（依速率調整），Back-drill 深度一致性。
EQ
等化器 PVT 角落驗證在 FF/SS/SF/FS + 電壓 ±10% + 溫度全範圍下確認等化器係數收斂。
EQ
DFE tap 範圍足夠H1 tap 係數範圍能涵蓋最差通道（最長走線 + 最多過孔）的後游標 ISI。
CLK
PLL 鎖定範圍VCO tuning range 覆蓋所有 PVT 變化下的目標頻率範圍。
CLK
Jitter 規格符合TX Jitter < 規格（如 PCIe Gen5 TX TJ < 0.3 UI）；RX Jitter Tolerance > 規格。
PWR
PDN 阻抗仿真SerDes 供電軌 PDN 阻抗目標值 < 10 mΩ（1 MHz–1 GHz 帶內）。
PWR
AVDD 雜訊量測上板後量測 AVDD 雜訊，確認 SSN peak < 5 mVpp（依製程建議）。
FW
Link Training 完整測試涵蓋 Cold Reset / Warm Reset / ASPM 週期，確認 LTSSM 無 hang 狀態。
FW
壓力 BER 測試以 BERT 連跑 24h，確認 BER < 10⁻¹²；溫度 / 電壓邊界下重複測試。

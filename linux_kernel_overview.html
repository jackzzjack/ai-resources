Linux Kernel 分層架構
Linux Kernel 採用 單體核心（Monolithic Kernel） 架構，但透過模組化設計（LKM）與嚴格的子系統邊界，兼顧彈性與效能。從 User Space 的系統呼叫介面，到最底層的硬體抽象層，所有程式碼都執行於相同的特權空間（Ring 0）。
User Space（使用者空間）
Applications glibc / musl systemd / init Shell / CLI Tools Containers (runc, containerd)
System Call Interface（系統呼叫介面）— 核心邊界
syscall table seccomp / BPF filter ptrace / audit VDSO / vvar
核心子系統（Core Subsystems）
記憶體管理 MM 行程排程 SCHED 虛擬檔案系統 VFS 網路堆疊 NET IPC 安全模組 LSM Block Layer eBPF Runtime 電源管理 PM io_uring
裝置驅動層（Device Drivers & Driver Model）
字元裝置 char 區塊裝置 block 網路裝置 netdev GPU / DRM USB / PCIe / I2C / SPI IOMMU Rust Drivers（新）
架構相依層（arch/）— Hardware Abstraction
x86 / x86_64 ARM64 (aarch64) RISC-V MIPS / PowerPC / s390 LoongArch（新） Hardware（CPU / RAM / 周邊）
核心元件詳覽
⏱
行程排程器（Process Scheduler）
kernel/sched/

決定哪個行程或執行緒在何時使用 CPU。現行主排程器為 CFS（Completely Fair Scheduler），基於紅黑樹排序虛擬執行時間（vruntime）。6.6 起引入可插拔排程框架 sched_ext（BPF 排程器），允許使用者以 eBPF 程式替換排程邏輯。

CFS RT Sched Deadline Sched sched_ext NUMA-aware
💾
記憶體管理（Memory Management）
mm/

管理虛擬記憶體空間、實體頁框分配、交換空間（swap）、頁面回收（kswapd）及 NUMA 拓撲。分配器包含 Buddy System（頁框層）、SLAB/SLUB/SLOB（物件層）與 vmalloc（連續虛擬空間）。

Buddy Allocator SLUB THP mmap / mlock CXL Memory
📁
虛擬檔案系統（VFS）
fs/ · include/linux/fs.h

統一的檔案系統抽象層，定義 inode / dentry / file / super_block 四大物件。底層可掛載 ext4、XFS、Btrfs、F2FS、EROFS 等真實檔案系統，或 procfs、sysfs、tmpfs 等偽檔案系統。

ext4 Btrfs XFS tmpfs Page Cache
🌐
網路堆疊（Networking Stack）
net/ · drivers/net/

完整的 TCP/IP 協定棧，從 L2 乙太網路到 L7 套接字介面。包含 Netfilter（封包過濾/NAT）、TC（Traffic Control）、XDP（eXpress Data Path） 與 IPVS。支援 IPv4/IPv6、MPTCP、QUIC offload。

TCP/IP XDP Netfilter MPTCP nftables
🔌
eBPF 執行環境
kernel/bpf/ · include/linux/bpf.h

在核心中安全執行使用者定義位元組碼的沙盒虛擬機器。透過 JIT 編譯器轉為原生機器碼，並由驗證器（Verifier）確保安全性。廣泛用於可觀測性、網路加速、安全稽核與排程。

BPF Verifier JIT BTF CO-RE bpftool
🔐
Linux 安全模組（LSM）
security/

可插拔安全框架，允許多個安全策略模組同時運行（LSM Stacking）。內建 SELinux（強制存取控制）、AppArmor（路徑式 MAC）、Landlock（應用程式沙盒）、SMACK、Yama。

SELinux AppArmor Landlock seccomp integrity (IMA)
💿
區塊層（Block Layer）
block/ · drivers/block/

統一管理所有區塊裝置 I/O。現行採用 Multi-Queue Block Layer（blk-mq） 架構，支援高並發 NVMe SSD。I/O 排程器包含 mq-deadline、BFQ（Budget Fair Queueing）、Kyber。

blk-mq NVMe BFQ dm / LVM io_uring
🔗
行程間通訊（IPC）
ipc/ · kernel/signal.c

提供行程間交換資料與同步的機制，包含 Pipe / FIFO、Signal、System V IPC（semaphore/shm/msg）、POSIX MQ、Unix Domain Socket 與現代化的 io_uring 非同步介面。

Pipe Signal Shared Memory Unix Socket
⚡
電源管理（Power Management）
drivers/cpufreq/ · kernel/power/

管理系統、CPU 與裝置的電源狀態，涵蓋 CPUfreq（動態頻率調整）、CPUIdle（深度睡眠狀態）、Runtime PM（裝置按需休眠）與 ACPI / PSCI 平台介面。

CPUfreq CPUIdle ACPI schedutil Energy Model
🔭
核心追蹤與除錯（Tracing & Debug）
kernel/trace/ · tools/perf/

提供核心可觀測性的基礎設施，包含 ftrace（函式追蹤）、kprobes / uprobes（動態探針）、tracepoints（靜態探針）、perf（效能事件計數）。eBPF 已成為主流觀測工具（bpftrace, BCC）。

ftrace kprobes perf KASAN syzkaller

當前研究熱點（2024–2026）
以下議題是目前核心社群（kernel.org、LKML、LPC、Linux Plumbers Conference）最活躍的討論與開發方向，反映工業界與學術界的共同關注。
🔥 最熱門
Rust in Linux（核心 Rust 化）
自 6.1 起合入 Rust 基礎設施，目前 Rust 可用於撰寫驅動程式與核心模組。主要優勢是記憶體安全——消除 use-after-free、buffer overflow 等 C 語言常見漏洞，佔核心 CVE 的 70%+。Google、微軟、Meta 積極貢獻。
rust/ drivers/gpu/nova (Rust) rust-for-linux.com
🔥 最熱門
eBPF 擴展（BPF Superpowers）
eBPF 已成為可觀測性、網路加速、安全稽核、甚至排程（sched_ext）的統一框架。研究方向包含：BPF token（容器化授權）、BPF struct_ops 擴展、BPF arena（大型共享記憶體）與 bpf_lsm 整合。
kernel/bpf/ SCHED_EXT bpftrace
⚡ 熱門
PREEMPT_RT 實時核心
歷時 20 年的實時補丁終於在 6.12 完整合入主線。將所有自旋鎖替換為可睡眠 mutex，達成完全可搶佔，延遲可低至數十微秒。對工業控制、金融交易、車載系統意義重大。
PREEMPT_RT kernel/locking/ rt-tests
⚡ 熱門
io_uring 非同步 I/O
由 Jens Axboe 主導，透過共享環形緩衝區大幅降低系統呼叫開銷，在高並發網路與儲存場景效能超越 epoll。io_uring 也進一步擴展為通用非同步執行框架，支援網路操作（send/recv）、行程管理等。
io_uring.c liburing IORING_OP_*
⚡ 熱門
CXL（Compute Express Link）
基於 PCIe 5.0 的新型互連技術，允許 CPU 與加速器（GPU、FPGA、Smart NIC）共享記憶體池（Memory Pooling）。Linux 正積極整合 CXL 1.1/2.0/3.0，涉及 NUMA、DAX、pmem 等多個子系統協作。
drivers/cxl/ linux-cxl@vger QEMU CXL
💡 上升中
機密運算（Confidential Computing）
Intel TDX（Trust Domain Extensions）、AMD SEV-SNP、ARM CCA 等硬體可信執行環境（TEE）。Linux 提供 TDX Guest 支援、KVM 整合與 remote attestation 機制，讓雲端租戶程式碼對 Hypervisor 不可見。
arch/x86/coco/ AMD SEV Intel TDX
💡 上升中
記憶體安全強化（Memory Safety）
除 Rust 引入外，C 端也在強化：CFI（Control Flow Integrity）、KCFI（核心 CFI）、PAC（Pointer Authentication on ARM）、MTE（Memory Tagging Extension）、KFENCE（低開銷記憶體錯誤偵測）。
KCFI KFENCE KASAN
💡 上升中
能源效率與綠色運算
Linux 的 Energy Aware Scheduling（EAS）讓排程器感知 CPU 能耗模型，在 ARM big.LITTLE 架構上優先使用小核。Intel EPP（Energy Performance Preference）、運算域感知電源管理（Per-die PM）也是熱門方向。
EAS schedutil powercap/
🔭 長期議題
RISC-V 生態完善
RISC-V 架構在嵌入式與伺服器領域快速成長，Linux 持續完善 RISC-V 支援：Svnapot（Huge TLB）、Zicbom（Cache Management）、KVM/H 擴展（Hypervisor）與 Vector 擴展（SIMD）。
arch/riscv/ linux-riscv@lists
🔭 長期議題
核心隔離與容器安全
Namespace、Cgroups v2、Seccomp、Landlock 的組合提供更細緻的容器隔離。研究方向包含：減少 namespace escape 攻擊面、unprivileged container 支援、io_uring 的 cgroup 整合與記憶體壓力回調。
cgroups v2 Landlock userns
近期版本特色（6.1 – 6.14）
Linux Kernel 約每 8–10 週發布一個主要版本，由 Linus Torvalds 主持。版本號格式為 6.x（自 5.20 直接跳為 6.0）。每年約有一個版本被選為 LTS（Long-Term Support），維護週期 6 年。
v6.14 2025 Q2 stable
最新穩定版 — AI 加速與硬體新支援
AMD RDNA 4（RX 9000 系列）初步 GPU 支援（amdgpu）
Intel Panther Lake / Arrow Lake S 平台支援
sched_ext BPF 排程器進一步穩定，加入更多 ops
io_uring 網路操作（zerocopy send）增強
Rust 抽象層擴展：更多 platform bus、PCI、GPIO 綁定
ACPI PPTT v3 拓撲感知（ARM CPU cache hierarchy）
v6.13 2025 Q1 stable
記憶體管理大躍進 + Rust 驅動擴展
MGLRU（Multi-Gen LRU）頁面回收全面預設啟用
Maple Tree 完全取代 VMA 紅黑樹，mmap 效能大幅提升
Rust：nova 驅動（NVIDIA GSP-only 開源驅動原型）
sched_ext scx 介面穩定，Meta/Google 開始在生產環境部署
ARM64 PMU（效能監控）多核心改進
v6.12 2024 Q4 LTS
🏆 LTS 版本 — PREEMPT_RT 完整合入歷史里程碑
PREEMPT_RT 全面合入主線：歷時 20 年的 realtime 補丁正式進入 upstream，選項為 CONFIG_PREEMPT_RT
sched_ext（可插拔 BPF 排程器）正式合入主線
bcachefs 檔案系統進入 mainline（5.1 後最重要的新 FS）
ARM64 Permission Overlay Extension（POE）記憶體保護
Intel FRED（更快速的例外/中斷入口）
AMD 下一代 IOMMU v2 改善
v6.11 2024 Q3 stable
IoT / 嵌入式 + 安全強化
Landlock 網路存取控制（限制行程的 TCP bind/connect）
mseal() 系統呼叫：記憶體區段密封，防止 mprotect 竄改
AMD IOMMU v2：提升 GPU 直通的虛擬化效能
Rust 工具鏈升級至 1.80，新增更多 kernel API 綁定
RISC-V：Sv57 五層頁表支援（128 PB 虛擬位址空間）
v6.10 2024 Q2 stable
GPU 進化 + 時間命名空間
nouveau（NVIDIA 開源驅動）開始支援 GSP 韌體（Turing+ GPU）
Time Namespace：容器可擁有獨立系統時鐘偏移
AMD Zen 5 / Strix Point APU 基礎支援
Intel Xe2 GPU 驅動（Battlemage）初步支援
BPF：Type Matching（更彈性的 BPF-to-kernel 型別驗證）
v6.9 2024 Q2 stable
網路加速 + AI 基礎設施
AMD P-State EPP（Energy Performance Preference）自動調頻改進
virtio-net 吞吐量提升，支援 NIC 熱插拔改進
BPF：BPF arena（大型連續共享記憶體，跨 user/kernel）
RISC-V：T-Head 廠商擴展、SBI 2.0 支援
Intel IFS（In-Field Scan）CPU 診斷支援
v6.8 2024 Q1 stable
Rust 網路驅動里程碑
第一個 Rust 網路驅動進入上游（Asix USB 網路卡）
Intel Meteor Lake 平台完整支援（EDP 顯示、NPU 初步）
cachestat() 系統呼叫：查詢頁面快取命中狀態
drm/panthor：ARM Mali Valhall/G610 開源驅動合入
FUSE passthrough：容器化 FUSE 效能大幅提升
v6.6 2023 Q4 LTS
🏆 LTS — MGLRU + sched_ext 預覽
MGLRU 預設啟用（Multi-Gen LRU，Android/Meta 長期使用的頁面回收算法）
sched_ext 合入（BPF 可插拔排程器，首個上游版本）

Shadow Stack（x86 SHSTK）：防止 ROP 攻擊
AMD TLB 效能改進 on EPYC Genoa/Bergamo
v6.1 2022 Q4 LTS
🏆 LTS — Rust 正式進入核心
Rust 語言基礎設施首次合入主線（rust/ 目錄，含編譯基礎、macro、core bindings）
MGLRU 首次合入（但尚未預設啟用）
Maple Tree：取代 VMA 管理的紅黑樹，提升 mmap 並發安全性
AMD RDNA 3（RX 7000）初步 GPU 支援
LoongArch 架構正式加入（龍芯 3A5000+）

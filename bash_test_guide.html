BASH · SHELL SCRIPT

test：讓腳本會判斷的語法

把條件判斷想成只回答「真」或「假」的守門員。它依檔案、字串或數字的狀態，決定腳本下一步走哪條路。

心智模型三種寫法運算子流程陷阱

先懂這個：真其實回傳 0

提出問題

檔案存在嗎？變數是空的嗎？數字是否大於 10？

test 判斷

條件為真時退出狀態為 0；為假時是非 0。

if 分流

if 看的是前一個命令的退出狀態，不是文字 true 或 false。

採取動作

依結果執行 then、elif 或 else。

三種外觀，同一個核心

if [ "$name" = "Jack" ]; then echo "Hello, Jack" fi # [ 與 ] 都是參數，兩側必須留空白。
if [[ $name == J* && -n $name ]]; then echo "以 J 開頭且不是空字串" fi # Bash 建議優先：支援 && 與模式比對。
if test -f "$file"; then echo "這是一個一般檔案" fi # [ "$file" -f ] 是 test 的另一種寫法。
你可以這樣想：[ 是 test 的別名；[[ 是 Bash 關鍵字，功能更完整。Bash 腳本優先用 [[ ... ]]；需要 POSIX sh 相容性才用 [ ... ]。

把問題配對到正確運算子

問題運算子例子
字串相等= / ==[[ $role == admin ]]
空或非空-z / -n[[ -n $token ]]
整數比較-eq -ne -gt -ge -lt -le[[ $count -ge 3 ]]
檔案與目錄-e -f -d[[ -f $config ]]
權限-r -w -x[[ -x ./deploy.sh ]]
邏輯判斷&& || ![[ -f $f && -s $f ]]

安全檔案檢查：從輸入到動作

01 INPUT
取得路徑
file=${1:-}
02 GUARD
先檢查有輸入
[[ -z $file ]]
03 TEST
確認一般檔案
[[ -f $file ]]
04 ACTION
處理或回報錯誤
file=${1:-} if [[ -z $file ]]; then echo "用法：$0 <檔案路徑>"; exit 2 elif [[ -f $file ]]; then echo "可處理：$file" else echo "不是一般檔案：$file"; exit 1 fi

常見誤解：看起來很像，結果不同

踩雷寫法原因改法
[ $name = Jack ]空值或空白會改變參數數量。[ "$name" = "Jack" ] 或 [[ $name == Jack ]]
[$x -gt 3]缺空白會變成找不到的命令。[ "$x" -gt 3 ]
[ $n > 3 ]> 可能被當成重導向。[[ $n -gt 3 ]]
[ $a -a $b ]舊式 AND 的優先序不易閱讀。[[ $a && $b ]]

30 秒檢核表

□ Bash 腳本優先選 [[ ... ]]
□ 數字用 -eq/-gt，不是 >
□ [ ... ] 的變數加引號、括號兩側留空白
□ 分清 -e、-f、-d
□ 失敗分支提供訊息與 exit code
□ 執行 shellcheck script.sh 檢查
